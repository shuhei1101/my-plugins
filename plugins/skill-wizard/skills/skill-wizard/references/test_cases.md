# テストケース詳細

対話形式スキルのテストケース生成時に参照する、正常系・異常系の詳細リスト。

## 正常系テストケース（5〜7件）

1. **基本フロー**: 全ステップを順番に実行し、想定通りの出力が得られるか
2. **スキップ機能**: 起動時に追加情報を提供し、特定ステップが適切にスキップされるか
3. **サブエージェント提案**: サブエージェントからの提案・推奨選択肢が生成されるか
4. **コンテキスト引き継ぎ**: 前のステップの情報が次のステップに正しく引き継がれるか
5. **セッション履歴**: 各ステップの実行内容が正しく記録されるか
6. **条件分岐**: 選択肢に応じた分岐が正しく機能するか
7. **ループ処理**: 繰り返しが必要なステップが正しく動作するか

## 異常系テストケース（8〜10件、正常系より多め）

1. **選択肢外入力**: 選択肢に該当しない入力をしたとき、適切に「その他」判定またはエラーメッセージが出るか
2. **必須入力未提供**: 必須の入力項目を空白または未提供で進めようとしたとき、入力を促すか
3. **無効なファイルパス**: 存在しないファイルパスを指定したとき、エラーハンドリングされるか
4. **不完全な情報**: スキル概要が曖昧・不足している状態で進行しようとしたとき、追加情報を求めるか
5. **フロー外への脱線**: 選択肢で許可されていない操作を試みたとき、確認または再選択が促されるか
6. **矛盾した選択**: 前ステップと矛盾する選択をしたとき、警告または確認が出るか
7. **空白入力**: 入力フィールドに空白のみを入力したとき、適切に処理されるか
8. **セッション履歴エラー**: セッション履歴が破損・欠損している場合の再開処理が適切か
9. **ステップスキップの誤判定**: スキップ条件を満たさないのにスキップされないか（条件の厳格性）
10. **無限ループ**: ループ条件が不適切で無限ループに陥らないか

## テスト結果ファイル形式

各テストケースの実行結果は以下の形式で記録する：

```markdown
# テストケース: T03 - 選択肢外入力

## テストシナリオ
選択肢に該当しない入力をして、適切に処理されるかを確認

## 実行内容

### ステップ1.1: メインメニュー
- 入力: 「スキルのテストをしたい」（選択肢外）
- 出力: 「申し訳ありませんが、以下から選択してください...」
- 選択: その他 → テストを実行
- 判定: ✅ OK（選択肢外の入力が適切に処理された）

### ステップ1.7: テストケース生成
- 入力: -
- 出力: テストケース一覧（15件生成）
- 選択: このテストケースでOK
- 判定: ✅ OK

## 総合判定: ✅ PASS

期待通りに動作し、選択肢外入力が適切に「その他」として処理された。
```

## 注意事項

- テストケースの観点は対象スキルの複雑さに応じて調整する
- 特に分岐が多いスキルやサブエージェント起動が複雑なスキルは異常系を重点的に
- 実際のテストケース数は対象スキルの規模に応じて増減可能
