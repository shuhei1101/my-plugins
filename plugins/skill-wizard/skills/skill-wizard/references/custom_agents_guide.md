# カスタムサブエージェント ガイド

スキル専用のカスタムサブエージェントを作成・管理するためのガイド。

---

## 概要

スキル内に `agents/` フォルダを設置することで、そのスキル専用のサブエージェントを定義できる。

- `/skill-creator` の `agents/` 構造を基本として踏襲する
- **加えて**、フロントマターを必ず記述する（将来の正式エージェント格上げに備えるため）
- 現時点では「指示書として読み込んで使う」形でも「正式エージェントとして呼び出す」形でも使える

---

## ディレクトリ構造

```
my-skill/
├── SKILL.md
└── agents/
    ├── persona-a.md
    ├── persona-b.md
    └── analyzer.md
```

## エージェントファイルの書き方

### フロントマター（必須）

```yaml
---
name: agent-slug
description: このエージェントの用途・呼び出し文脈を1文で説明する。いつ使われるかを明確に書く。
model: claude-haiku-4-5-20251001
---
```

#### フロントマターのフィールド

| フィールド | 必須 | 説明 |
|---|---|---|
| `name` | ✓ | エージェントのスラッグ（kebab-case） |
| `description` | ✓ | 用途と呼び出し文脈。いつ使うかを明示する |
| `model` | 推奨 | 使用モデル。軽量タスクは `claude-haiku-4-5-20251001`、複雑なタスクは `claude-sonnet-4-6` |
| `tools` | 任意 | 許可するツール一覧 |
| `disallowedTools` | 任意 | 禁止するツール一覧 |

#### ⚠️ プラグイン内エージェントで使えないフィールド（セキュリティ境界）

- `hooks`
- `mcpServers`
- `permissionMode`

---

### ボディ（本文）

フロントマターの後に、エージェントへの指示を Markdown で記述する。

```markdown
---
name: my-analyzer
description: データを分析してレポートを生成する。「分析して」「まとめて」といった文脈でmy-skillから呼び出される。
model: claude-sonnet-4-6
---

# My Analyzer

あなたは[役割]です。[何を優先するか]を最優先します。

## コアバリュー
- ...

## 推論スタイル
- ...

## 回答形式

以下を提供してください:
1. ...
2. ...
```

---

## モデルの選び方

| タスクの種類 | 推奨モデル |
|---|---|
| 意見生成・ペルソナ | `claude-haiku-4-5-20251001`（軽量・高速） |
| 分析・評価・採点 | `claude-sonnet-4-6`（高精度） |
| 複雑な設計・アーキテクチャ | `claude-sonnet-4-6` または `claude-opus-4-6` |

## ペルソナエージェントのパターン

`/idea` スキルのペルソナを参考にした、意見生成型エージェントのパターン：

```markdown
---
name: cost-conscious
description: コスト重視の観点から意見を提供する。開発・運用コストの最小化を優先する。
model: claude-haiku-4-5-20251001
---

# コスト重視

あなたはコスト重視の実装者です。...

## コアバリュー
- ...

## 回答形式
1. 意見（1〜2文）
2. 根拠（2〜3文、コストの観点から具体的に）
```

## 分析・採点エージェントのパターン

`/skill-creator` の `agents/grader.md` を参考にした、採点・分析型エージェントのパターン：

```markdown
---
name: grader
description: 実行結果をアサーションに基づいて採点する。skill-creatorの評価フェーズで呼び出される。
model: claude-sonnet-4-6
---

# Grader

あなたは採点者です。...

## 入力
- expectations: 評価するアサーションのリスト
- transcript_path: 実行トランスクリプトのパス
- outputs_dir: 出力ファイルのディレクトリ

## プロセス
1. トランスクリプトを読む
2. 出力ファイルを検証する
3. 各アサーションを採点する
```

## SKILL.md からの参照方法

SKILL.md 内でエージェントファイルを参照する際の記述例：

```markdown
## サブエージェント利用

詳細設計フェーズでは以下のエージェントを活用する:

- `agents/analyzer.md` — 構造分析と改善提案（`read agents/analyzer.md` して指示を読み込む）
- `agents/reviewer.md` — レビューと品質チェック
```

または、将来的に正式エージェントとして呼び出す場合：

```bash
# プラグイン名:エージェント名 形式で呼び出し（プラグインが有効な場合）
@my-plugin:analyzer
```

## `/idea` ペルソナ との使い分け

| 状況 | アプローチ |
|---|---|
| 既存カテゴリで十分 | `/idea` を使う（implementation / docs / project / review / naming） |
| スキル専用のペルソナが必要 | スキルの `agents/` に独自ペルソナを作成 |
| `/idea` が使えない環境 | スキルの `agents/` のエージェントをサブエージェントとして直接起動 |
