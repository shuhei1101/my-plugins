# 既存スキル編集フェーズ

既存のスキルを編集する際に使用します。ステップの編集、追加、削除、並び替えを行います。

---

## ステップ4.1(編集対象の特定)

**条件**: 編集フロー開始時、またはスキル名と修正内容が直接指定された場合（メインメニューをスキップして直接このステップから開始可能）

**入力**: 編集対象のスキルパスまたはスキル名（修正内容が明示されている場合はそれも含む）

**AI実施タスク**:
  - 対象スキルのSKILL.mdを読み込む
  - ステップ構造を解析して一覧化する
  - **修正内容が既に指定されている場合**（例: 「file-summaryのdescriptionを修正して」）:
    - 該当箇所（ステップ、セクション、フィールド）を特定する
    - ステップ4.2に直接進み、該当箇所の編集を開始する
    - memoryには修正対象を明記する
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_edit.md` を作成:
    ```
    # スキル編集セッション - {YYYYMMDD} {HHMMSS}
    
    ## 開始時刻
    {時刻}
    
    ## 編集対象
    {スキル名} ({パス})
    
    ## 現在のステータス
    ステップ4.1: 編集対象の特定
    
    ## 編集内容
    {修正内容が指定されている場合は「{対象}: {修正内容}」、未定の場合は「（未定）」}
    ```

**出力**: 
  - **修正内容が未指定の場合**: 対象スキルのステップ一覧と概要を表示
  - **修正内容が既に指定されている場合**: 該当箇所の現在の内容を表示し、ステップ4.2へ進む

**選択肢**（修正内容が未指定の場合のみ提示）:
  1. 特定のステップを編集する（番号で指定）
  2. ステップの追加・削除・並び替えを行う
  3. スキル全体の概要・前提知識部分を修正する
  4. その他（入力してください）

**補足**: 
  - 修正内容が既に指定されている場合（例: 「file-summaryのdescriptionを修正して」）は、この選択肢をスキップしてステップ4.2に直接進みます
  - メインメニュー（ステップ1.1）を経由せず、直接編集依頼から開始することも可能です

---

## ステップ4.2(ステップ編集)

**条件**: 編集対象のステップが指定された場合

**入力**: -

**AI実施タスク**:
  - 対象ステップの現在の内容を表示する
  - 改善案を複数の観点から提案する
    - **`/idea` を使用**して改善案を生成
      - トピック: 「{対象ステップ}の改善案」
      - 観点: 簡潔さ・網羅性・ユーザー体験等（ステップの性質に応じて自動選択）
    - **`/idea` が利用できない場合**:
      - サブエージェント: 2〜3体を並列実行
      - ペルソナが無い場合: 自分で観点を設定（「簡潔さ重視」「網羅性重視」「ユーザー体験重視」など）
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_edit.md` を更新:
    ```
    ## 編集内容
    ### ステップ{番号}: {ステップ名}
    #### 変更前
    {現在の内容}
    
    #### 改善提案
    1. {提案1}
    2. {提案2}
    ```

**出力**: 現在のステップ内容 + 改善提案

**選択肢**:
  1. 修正内容を伝える
  2. 提案された改善を採用する（番号で指定）
  3. このステップは変更なし → 一覧に戻る
  4. その他（入力してください）

**補足**: 編集完了後は「編集対象の特定」ステップに戻り、他のステップの編集を続けられる。

---

## ステップ4.3(編集内容の確認)

**条件**: ユーザーが編集完了を宣言した場合

**入力**: -

**AI実施タスク**:
  - 変更箇所を差分形式で表示する
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_edit.md` を更新:
    ```
    ## 最終編集内容
    - ステップ{番号}: {変更概要}
    - ステップ{番号}: {変更概要}
    ```

**出力**: 変更差分の一覧

**選択肢**:
  1. この内容でSKILL.mdを更新する
  2. 追加で修正する → 編集対象の特定へ
  3. 変更を破棄する
  4. その他（入力してください）

---

## ステップ4.4(SKILL.md更新)

**条件**: 更新承認済み

**入力**: -

**AI実施タスク**:
  - SKILL.mdの該当箇所を更新する
  - **プラグインバージョンを更新する**（必須）:
    - `{プラグインディレクトリ}/.claude-plugin/plugin.json` の `version` をインクリメント（パッチ: 軽微な修正、マイナー: 機能追加・ステップ変更）
    - 親マーケットプレースの `marketplace.json` の該当プラグインの `version` も同じ値に更新する
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_edit.md` を更新:
    ```
    ## 完了情報
    ### 更新時刻
    {時刻}
    
    ### 更新ファイル
    {パス}
    
    ### 変更ステップ数
    {数}
    ```

**出力**: 更新完了報告

**選択肢**:
  1. 完了 → プラグイン反映の確認へ
  2. 更新内容を確認する
  3. テストを実行する → ステップ5.1（テストフェーズ）へ
  4. さらに編集を続ける → 編集対象の特定へ
  5. その他（入力してください）

**補足（キャッシュ同期）**: 完了選択時、`workflow_guide.md` の「プラグインキャッシュ同期」に従い、プラグイン反映の選択肢を提示すること。

**補足**: 編集後は必ずテスト実行を推奨します。ステップ5.1でスキル構造の検証とテストケース実行が可能です。
