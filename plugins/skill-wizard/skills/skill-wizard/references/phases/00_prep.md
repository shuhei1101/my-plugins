# 事前準備フェーズ

このフェーズでは、スキル作成の方向性を決定し、必要な情報を収集します。

---

## ステップ1.1(メインメニュー)

**条件**: スキル起動時。ただし、以下の条件に該当する場合は該当ステップに直接ジャンプする（このステップをスキップ）

**入力**: ユーザーのリクエスト全文

**AI実施タスク**: 
- **起動時のコンテキスト分析**（このステップを実行するか判定）:
  
  1. **編集リクエストの検出** - 以下のパターンに該当する場合、ステップ4.1へジャンプ:
     - 「{スキル名}の{対象}を修正/変更/編集して」
     - 「{スキル名}のステップ{番号}を{操作}」
     - 「{スキル名}の改善案教えて」
     - 例: "file-summaryのdescriptionを修正して"、"ideaスキルのステップ2.1を追加して"
     → ステップ4.1（編集対象の特定）に直接ジャンプし、~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_edit.mdを作成
  
  2. **ランダム生成リクエストの検出** - 以下のパターンに該当する場合、ステップ1.2へジャンプ:
     - 「ランダムスキル生成」「ランダムで案出して」「何作るか迷ってる」
     - 例: "ランダムスキル生成したい"
     → ステップ1.2（ランダムスキル案生成）に直接ジャンプ
  
  3. **スキル概要提供済みの検出** - 以下のパターンに該当する場合、ステップ1.3へジャンプ:
     - 「スキル作成したい。要件は{概要}」「{具体的なスキル概要}を作って」
     - リクエスト内にスキルの目的・機能が明示されている
     - 例: "スキル作成したい。要件は、READMEファイルを自動生成するスキルで..."
     → ステップ1.3（スキルタイプの確認）に直接ジャンプし、スキル概要を抽出
  
  4. **テスト/チェックリクエストの検出** - 以下のパターンに該当する場合、ステップ5.1へジャンプ:
     - 「{スキル名}をテスト」「{スキル名}の構造チェック」
     → ステップ5.1（テストフェーズ）に直接ジャンプ
  
  5. **カスタムサブエージェント調整リクエストの検出** - 以下のパターンに該当する場合、ステップ6.1へジャンプ:
     - 「{スキル名}のエージェントを調整」「{スキル名}にサブエージェントを追加」
     - 「カスタムエージェントを作りたい」「agents/フォルダを整理したい」
     - 例: "ideaのペルソナエージェントを追加して"、"skill-wizardのエージェントを更新して"
     → ステップ6.1（カスタムサブエージェント管理）に直接ジャンプ（スキル名を引き継ぐ）
  
  6. **上記のいずれにも該当しない場合**: 通常のメインメニュー処理を実行

- **通常のメインメニュー処理**（上記1-4に該当しない場合のみ実行）:
  - スキル作成セッションの開始
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_creation.md` を作成し、セッション開始を記録:
  ```
  # スキル作成セッション - {YYYYMMDD} {HHMMSS}
  
  ## 開始時刻
  {時刻}
  
  ## 現在のステータス
  ステップ1.1: メインメニュー
  
  ## スキル情報
  （未定）
  ```

**出力**: 
  - このスキルの説明（対話形式スキル・単発スキルの新規作成・既存編集・カスタムサブエージェント管理ができること）

**選択肢**:
  1. スキル概要を入力して新規作成する
  2. 既存のスキルを編集する（スキル指定してください）
  3. 既存のスキルの構成をチェック・テストする（スキル指定してください）
  4. 🎲 ランダムでスキル案を生成する（何を作るか迷ったらこれ）
  5. スキルのカスタムサブエージェントを調整する（スキル指定してください。未指定は skill-wizard）
  6. その他（入力してください）

**補足**: 
  - **このステップは条件付きで実行される**: 起動時のコンテキスト分析により、該当フローが特定された場合はこのステップをスキップ
  - **ジャンプ先の詳細**:
    - 編集リクエスト → ステップ4.1（編集対象の特定）
    - ランダム生成リクエスト → ステップ1.2（ランダムスキル案生成）
    - スキル概要提供済み → ステップ1.3（スキルタイプの確認）
    - テスト/チェックリクエスト → ステップ5.1（テストフェーズ）
    - カスタムサブエージェント調整 → ステップ6.1（カスタムサブエージェント管理）
  - **選択肢による遷移**:
    - 選択肢1 → ステップ1.3（スキルタイプの確認）
    - 選択肢2 → ステップ4.1（編集対象の特定）
    - 選択肢3 → ステップ5.1（テストフェーズ）
    - 選択肢4 → ステップ1.2（ランダムスキル案生成）
    - 選択肢5 → ステップ6.1（カスタムサブエージェント管理）

## ステップ1.2(ランダムスキル案生成)

**条件**: 
  - メインメニューで選択肢4が選択された場合
  - または、起動時リクエストに「ランダム生成」の意図が含まれていた場合（ステップ1.1のコンテキスト分析により直接ジャンプ）

**入力**: -

**AI実施タスク**:
  - この機能はおまけ機能。詳細実装・出力形式・分岐ルールは `references/bonus_random_ideas.md` を参照

**出力**:
  - 「スキル案を生成しました！」
  - スキル案の一覧を表示
  - 「気になる案があれば番号を選んでください。そのスキルを一緒に作りましょう！」

**選択肢**:
  1. 気になるスキル案を選ぶ（番号で指定） → ステップ1.3（スキルタイプの確認）へ
  2. もう一度ランダム生成する → このステップを再実行
  3. 自分でスキル概要を入力する → ステップ1.3（スキルタイプの確認）へ
  4. その他（入力してください）

**補足**: 
  - このステップの詳細は `references/bonus_random_ideas.md` に集約して管理する

---

## ステップ1.3(スキルタイプの確認)

**条件**: メインメニューで「スキル概要を入力して新規作成する」が選択された場合、またはランダム生成からスキル案が選ばれた場合

**入力**: スキル概要（既に提供されている場合）

**AI実施タスク**:
  - スキル概要が未提供の場合、入力を促す
  - スキル概要から、対話形式スキルか単発スキルかを判定する
    - 対話形式スキル: ステップバイステップでユーザーと対話しながら進める形式（例: skill-wizard、rfp）
    - 単発スキル: 1回のリクエストで完結する形式（例: idea、コード生成、要約）
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_creation.md` を更新:
    ```
    ## スキル情報
    ### 概要
    {ユーザーが入力した概要}
    
    ### スキルタイプ判定
    {対話形式 / 単発}
    理由: {判定理由}
    ```

**出力**:
  - スキル概要の確認
  - 判定結果の説明:
    - 「このスキルは【対話形式スキル / 単発スキル】と判定しました」
    - 判定理由を簡潔に説明

**選択肢**:
  1. この判定でOK → 対話形式ならステップ1.7へ、単発ならステップ1.4へ
  2. 判定を変更する（対話形式 / 単発を指定）
  3. スキル概要を修正する → このステップを再実行
  4. その他（入力してください）

**補足**:
  - 判定が難しい場合は、ユーザーに確認する
  - 対話形式スキルの場合: このskill-wizardで設計を進める
  - 単発スキルの場合: `/skill-creator` への移譲を試みる（ステップ1.4へ）
    - skill-creatorが利用できない場合は、インストールを促すか、このskill-wizardで対話形式として設計する選択肢を提示する

---

## ステップ1.4(skill-creatorへの移譲)

**条件**: スキルタイプが「単発スキル」と確定した場合

**入力**: スキル概要、スキル名（あれば）、トリガーフレーズ（あれば）

**AI実施タスク**:
  - `/skill-creator` が利用可能かチェックする
  - 利用可能な場合:
    - サブエージェントで `/skill-creator` を起動
    - 必読情報: スキル概要、スキル名、トリガーフレーズ、その他ユーザーが提供した情報
    - タスク: 「以下の情報をもとに単発スキルを作成してください。スキル概要: {概要}、スキル名: {名前}、トリガーフレーズ: {トリガー}」
    - skill-creatorの実行結果（生成されたSKILL.md）を取得
  - 利用不可の場合:
    - 「`/skill-creator` が見つかりません。このskill-wizardで単発スキルを設計することもできますが、推奨は `/skill-creator` の利用です」

**出力**:
  - skill-creatorが利用可能な場合: 生成されたSKILL.mdの内容を表示
  - 利用不可の場合: その旨を通知

**選択肢**:
  1. この内容でOK → ステップ1.5（微調整）へ
  2. skill-creatorに追加情報を渡して再実行する
  3. skill-creatorが利用できないので、このskill-wizardで対話形式として設計する → ステップ1.7へ
  4. その他（入力してください）

**補足**:
  - skill-creatorは単発スキルの作成に特化しているため、単発スキルはそちらに移譲する方が効率的
  - skill-creatorが生成したSKILL.mdは、そのまま使えることが多いが、微調整が必要な場合はステップ1.5で対応

---

## ステップ1.5(単発スキル微調整)

**条件**: skill-creatorでスキルが生成され、微調整が必要な場合

**入力**: skill-creatorが生成したSKILL.md

**AI実施タスク**:
  - 生成されたSKILL.mdを確認する
    - descriptionの形式チェック: `XXXを行う。「」...「」という依頼で起動。`
    - 最初に提供された情報との整合性チェック（スキル概要、スキル名、トリガーフレーズ）
    - 言語チェック: 英語のみか、日本語が含まれているか
    - 不足している情報や改善点を洗い出す

**出力**:
  - チェック結果のサマリー:
    - description形式: OK / NG（修正案を提示）
    - 情報の整合性: OK / 不整合あり（詳細を提示）
    - 言語: 英語のみ / 日本語含む
    - その他の改善点（あれば）

**選択肢**:
  1. この内容でOK → SKILL.mdを出力する（ステップ3.4へ）
  2. 改善項目を選択する → ステップ1.6（改善項目選択）へ
  3. 修正内容を伝える → 修正して再提示
  4. skill-creatorに戻して再生成する → ステップ1.4へ
  5. その他（入力してください）

**補足**:
  - 微調整が不要な場合、このステップはスキップしてステップ3.4（SKILL.md出力）に直接進んでもよい
  - descriptionの形式は特に重要。トリガーフレーズが「」で囲まれ、「という依頼で起動。」で終わることを確認

## ステップ1.6(改善項目選択)

**条件**: ステップ1.5で「改善項目を選択する」が選択された場合、または対話形式スキルのレビュー時に改善項目の選択が必要な場合

**入力**: -

**AI実施タスク**:
  - 改善可能な項目をリストアップする
  - 各項目にデフォルトでチェック（✓）をつける
  - 対話形式スキルの場合:
    - `references/workflow_guide.md`、`references/test_cases.md`、（あれば）関連ガイドを読んで改善観点を抽出する
    - **`/idea` を使用**して、複数の観点（ユーザビリティ・明瞭性・分岐網羅性・運用性など）から改善案を追加生成する
    - `/idea` が利用できない場合はサブエージェントを直接起動して同等の観点で案出しする
  - 「その他」項目では `/idea` を使って追加の改善案を生成する準備をする
    - トピック: 「このスキルの改善案」
    - 観点: ユーザビリティ・明瞭性・拡張性等

**出力**:
  - 改善項目リスト（チェックリスト形式）は `assets/improvement_checklist_template.md` をテンプレートとして提示する

**選択肢**:
  1. この改善項目で実行する（編集後のリストを貼り付けてください）
  2. すべての改善を実行する
  3. 戻る → 直前のレビュー/確認ステップへ
  4. その他（入力してください）

**補足**:
  - 「その他の改善」が含まれている場合、`/idea` を起動して複数の観点から改善案を生成する
  - `/idea` が利用できない場合は、サブエージェントを直接起動して改善案を生成
  - 単発スキルの改善実行後はステップ1.5に戻り、結果を再チェックする
  - 対話形式スキルの改善実行後は、直前のレビュー/構造チェックステップに戻って再チェックする

## ステップ1.7(参考情報の収集)

**条件**: 対話形式スキルの新規作成フローで、スキルタイプ確認後

**入力**: -

**AI実施タスク**:
  - ワークスペース内の既存スキルを走査し、今回のスキル作成で参考になりそうなスキルやファイルを特定する
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_skill_creation.md` を更新:
    ```
    ### 参考情報
    - {参考スキル1}
    - {参考スキル2}
    - {参考ファイル1}
    ```

**出力**:
  - スキル作成の参考になりそうな情報の提案
  - 提案スキル: AIが現在のワークスペース内で読めるスキルから、関連性の高いものをピックアップして提示する

**選択肢**:
  1. 提案されたスキルを参考にする（追加があれば記載してください）
  2. 自分でファイルパスやスキルを指定する
  3. 参考情報なしで進める
  4. その他（入力してください）
