# カスタムサブエージェント管理フェーズ

スキル専用のカスタムサブエージェント（`agents/` フォルダ内のエージェントファイル）を新規作成・編集・タスクへの紐づけを行うフェーズ。

**前提**: `references/custom_agents_guide.md` を必ず参照すること。

---

## ステップ6.1(カスタムサブエージェント管理 — 対象確認)

**条件**: 
  - メインメニューで「カスタムサブエージェントの調整」が選択された場合
  - またはステップ2.6bからジャンプした場合（特定タスクのエージェント設定）

**入力**: 対象スキルのパスまたはスキル名（任意）、対象タスク名（ステップ2.6bから来た場合）

**AI実施タスク**:
  - 対象スキルを特定する（未指定の場合は skill-wizard を対象にする）
  - `{スキルパス}/agents/` の存在を確認し、既存のエージェントファイルを一覧化する
  - 対象スキルの SKILL.md を読み込み、サブエージェントを使っているステップを特定する
  - ステップ2.6bから来た場合: 対象タスクの詳細（条件・AIタスク・選択肢）を確認する
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_agent_management.md` を作成:
    ```
    # カスタムサブエージェント管理 - {YYYYMMDD} {HHMMSS}

    ## 対象スキル
    {スキル名} ({パス})

    ## 対象タスク（あれば）
    {タスク名}（ステップ2.6bから来た場合）

    ## 既存エージェント
    {一覧、なければ「なし」}

    ## サブエージェントを使っているステップ
    {一覧}
    ```

**出力**:
  - 対象スキルとその `agents/` 内のエージェント一覧
  - サブエージェントを使っているステップの一覧

**選択肢**:
  1. 新しいエージェントを作成する → ステップ6.2へ
  2. 既存のエージェントを編集する（名前またはファイル名で指定）→ ステップ6.2へ
  3. エージェントを特定のタスクに紐づける → ステップ6.3へ
  4. 別のスキルを対象にする（スキル名を指定してください）
  5. その他（入力してください）

**補足**:
  - ステップ2.6bから来た場合、対象タスクは自動的に確定しているため選択肢3に直接進むことも自然
  - `agents/` フォルダが存在しない場合は、作成することを提案する

---

## ステップ6.2(エージェント詳細設計)

**条件**: 新規作成または既存エージェント編集の場合

**入力**: エージェントの目的・役割（新規）またはファイルパス（編集）

**AI実施タスク**:
  - **新規作成の場合**:
    - エージェントの目的・役割・呼び出し文脈をヒアリングする
    - `/idea` のペルソナが参考になる場合は `agents/` の既存ファイルを参照する
    - `custom_agents_guide.md` のパターンを参考にエージェントファイルを設計する
    - フロントマター（name / description / model）のドラフトを提案する
    - ボディ（役割・コアバリュー・推論スタイル・回答形式）を設計する
  - **既存編集の場合**:
    - 現在のエージェントファイルを読み込む
    - 改善点や変更内容をヒアリングする
    - 変更案を提示する
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_agent_management.md` を更新:
    ```
    ## エージェント設計
    ### {エージェント名}
    #### フロントマター
    name: {slug}
    description: {description}
    model: {model}
    
    #### 役割
    {概要}
    ```

**出力**: エージェントファイルのドラフト（フロントマター + ボディ）

**選択肢**:
  1. この内容でOK → ステップ6.3（ファイル出力・タスク紐づけ）へ
  2. 修正内容を伝える → 修正して再提示
  3. モデルを変更する（haiku / sonnet / opus）
  4. ペルソナカテゴリを参考にしたい（implementation / docs / project / review / naming）
  5. エージェント一覧に戻る → ステップ6.1へ
  6. その他（入力してください）

**補足**:
  - ペルソナ型エージェントには `claude-haiku-4-5-20251001` が適切
  - 分析・採点型には `claude-sonnet-4-6` を推奨
  - descriptionには「いつ呼び出されるか」を明確に書くことが重要（トリガー文脈）

---

## ステップ6.3(エージェントファイル出力 & タスク紐づけ)

**条件**: エージェント設計が確定した場合、またはタスク紐づけが必要な場合

**入力**: -

**AI実施タスク**:
  - **ファイル出力**:
    - `{スキルパス}/agents/{slug}.md` にエージェントファイルを出力する
    - `agents/` フォルダが存在しない場合は作成する
  - **タスクへの紐づけ**（ステップ2.6bから来た場合、またはユーザーが紐づけを希望する場合）:
    - 「どのタスク（ステップ）にこのエージェントを紐づけますか？」を確認する（未確定の場合）
    - 対象タスクの `AI実施タスク` セクションに、エージェントの利用方法を追記する提案を生成:
      ```
      # 追記例（対象タスクのAI実施タスクに追加）
      - **カスタムエージェント活用**: `agents/{slug}.md` を読み込み、{役割}として並列起動する
        - 渡す情報: {コンテキスト}
        - 期待する出力: {出力形式}
      ```
    - SKILL.md または対応するwizardファイルへの反映を提案する
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_agent_management.md` を更新（完了情報）

**出力**: 出力されたファイルパスとタスク紐づけの差分提案

**選択肢**:
  1. このまま完了する → プラグイン反映の確認へ
  2. 別のタスクへの紐づけも追加する
  3. さらに別のエージェントを作成する → ステップ6.2へ
  4. エージェント一覧に戻る → ステップ6.1へ
  5. SKILL.mdへの反映を確認する → ステップ4.4（SKILL.md更新）へ
  6. その他（入力してください）

**補足（キャッシュ同期）**: 完了選択時、`workflow_guide.md` の「プラグインキャッシュ同期」に従い、プラグイン反映の選択肢を提示すること。

**補足**:
  - エージェントファイルを作成しただけではSKILL.mdには反映されない。必要に応じてステップ4.4で更新すること
  - タスク紐づけの際は、エージェントに渡すコンテキスト（スキル・ファイルパス・前のステップの出力など）を明確に記述することが重要
