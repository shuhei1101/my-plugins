# テストフェーズ

新規作成・編集の両方で利用可能。作成または編集したスキルの動作確認を行います。

---

## ステップ5.1(構造チェック)

**条件**: テストフェーズ開始時

**入力**: 対象スキルのSKILL.md

**AI実施タスク**:
  - SKILL.mdの行数をカウントする
  - 重複コンテンツの検出:
    - 同じ内容が複数箇所に記載されていないかチェック
    - 似た説明や冗長な記述を洗い出す
  - 推奨基準とのチェック:
    - 行数: 500行以下推奨（空白行を含む。`wc -l`や`Measure-Object -Line`でカウント）
    - description: トリガーフレーズ複数、「〜という依頼で起動」で終わる
    - 参照ファイル（もしあれば）: 300行超え（空白行含む）のファイルに目次があるか
  - 問題がある場合、改善提案を生成する

**出力**:
  - チェック結果のサマリー:

  | チェック項目     | 現状   | 推奨      | 判定 |
  | ---------------- | ------ | --------- | ---- |
  | 重複コンテンツ   | 3箇所  | なし      | ⚠️ |
  | SKILL.md行数     | 523行  | 500行以下 | ⚠️ |
  | description形式  | OK     | -         | ✅   |
  | 参照ファイル目次 | 未確認 | -         | -    |

  - 改善提案（問題がある場合のみ）:
    - 「重複: XXXの説明が2箇所にあります。「前提知識」セクションに統合することを推奨します」
    - 「XXXセクション（50行）をreferences/xxx.mdに分離することを推奨します」
    - 「descriptionにトリガーフレーズを追加してください: 「〜を作成して」「〜を設計して」等」

**選択肢**:
  1. このままテストを続行する
  2. 改善してからテストする → 編集フローへ
  3. その他（入力してください）

**補足**: 構造に問題があってもテストは実行可能だが、運用時の可読性やコンテキスト管理に影響する可能性がある。

## ステップ5.2(テストケース生成)

**条件**: 構造チェック完了後

**入力**: 対象スキルのSKILL.md

**AI実施タスク**:
  - 対象スキルの全ステップを分析し、テストケースを生成する
  - 正常系・異常系のテストケース詳細は `references/test_cases.md` を参照
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_test_run.md` を作成:
    ```
    # テスト実行 - {YYYYMMDD} {HHMMSS}
    
    ## 対象スキル
    {スキル名}
    
    ## テストケース数
    {数}
    
    ## テスト内容
    | ID  | 種別 | テスト内容 | 入力値（条件） | 期待結果 |
    | --- | ---- | ---------- | -------------- | -------- |
    （テストケース一覧）
    ```

**出力**: テストケース一覧を表形式で提示

  | ID  | 種別 | テスト内容     | 入力値（条件）                                 | 期待結果                                         |
  | --- | ---- | -------------- | ---------------------------------------------- | ------------------------------------------------ |
  | T01 | 正常 | 基本フロー実行 | 全ステップで標準的な回答を入力                 | 全ステップが順に実行され、SKILL.mdが出力される   |
  | T02 | 正常 | スキップ機能   | 起動時にスキル概要を含むリクエストを渡す       | 該当ステップがスキップされ、次ステップに進む     |
  | T03 | 異常 | 選択肢外入力   | 選択肢に存在しない番号や文字列を入力           | 「その他」に判定されるか、再入力が促される       |
  | ... | ...  | ...            | ...                                            | ...                                              |

**選択肢**:
  1. このテストケースでOK → テスト実行へ
  2. テストケースを追加・修正する
  3. その他（入力してください）

**補足**: テストケースの観点は対象スキルの複雑さに応じて調整する。特に分岐が多いスキルやサブエージェント起動が複雑なスキルは異常系を重点的に。

## ステップ5.3(テスト実行と結果確認)

**条件**: テストケース承認済み

**入力**: -

**AI実施タスク**:
  - 各テストケースをサブエージェントで並列実行（待つ）
    - サブエージェント: テストケース数分を並列起動
    - 必読情報: 対象スキルのSKILL.md、各テストケースのシナリオ
  - 各サブエージェントはテストシナリオに従ってスキルを実行し、以下を記録:
    - 各ステップで実行された内容
    - 選択した選択肢とその理由
    - 実際の出力内容
    - 期待結果との比較（OK / NG / WARN）
    - NG/WARNの場合は詳細な原因
  - 結果を `~/.claude/skill-test/{対象スキル名}/YYYYMMDD/test_case_{テストID}.md` に出力（ディレクトリが存在しない場合は自動作成）
  - 各サブエージェントの実行結果から `total_tokens`, `duration_ms` を収集する
  - テストケースごと・合計のトークン使用量を集計する
  - モデルの料金表に基づいてドル換算を算出する:
    - Opus 4: $15/M input, $75/M output（input/output不明の場合は中間値 $45/M で概算）
    - Sonnet 4: $3/M input, $15/M output（概算 $9/M）
    - Haiku 3.5: $0.80/M input, $4/M output（概算 $2.4/M）
  - `~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_test_run.md` を更新:
    ```
    ## テスト結果
    - OK: {数}件
    - NG: {数}件
    - WARN: {数}件
    
    ### 詳細
    （各テストケースの結果概要）
    
    ### トークン使用量
    （テストケースごとの集計表）
    ```

**出力**: テスト実行完了報告と結果サマリー

  ```
  テスト実行完了 (15件)
  - OK: 12件
  - NG: 2件
  - WARN: 1件
  
  📁 結果: ~/.claude/skill-test/{対象スキル名}/20260402/
  📁 記録: ~/.claude/skill-memory/skill-wizard/history/{YYYYMMDDHHMMSS}_test_run.md

  📊 トークン使用量
  | ID  | model  | tokens | time | cost   |
  | --- | ------ | ------ | ---- | ------ |
  | T01 | opus   | 45,230 | 38s  | ~$2.04 |
  | T02 | opus   | 12,100 | 12s  | ~$0.54 |
  | T03 | sonnet | 8,500  | 5s   | ~$0.08 |
  | ... | ...    | ...    | ...  | ...    |
  | 合計 |        | 65,830 | 55s  | ~$2.66 |
  ```

**選択肢**:
  1. 完了（結果を確認しました）
  2. スキルを修正する → 編集フローへ
  3. その他（入力してください）

**補足**: テスト結果ファイルの形式詳細は `references/test_cases.md` を参照。
