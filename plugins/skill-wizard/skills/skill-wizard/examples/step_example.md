# ステップ記述 — 具体例

対話形式スキルのステップは以下の形式で記述する。

---

## 具体例: Issue 追加フェーズのステップ

```markdown
## フェーズ2: 新規 Issue の追加

### ステップ2.3(選択肢の整理)

条件: ステップ2.2 完了後（連番採番済み）
入力: ステップ2.1 で得たタイトル・背景・選択肢候補

AI実施タスク:
  - ユーザーが選択肢を提示済み → そのまま整形
  - ユーザーが選択肢をスキップ → 背景から選択肢案を 2〜3 個生成し推奨方式も提案
  - assets/issue_format.md のフォーマットに従って整形

出力:
  以下の内容で `docs/Issues.md` に追記します。

  ## ISSUE-008: TTS プロバイダの選定 未決定

  **背景**: 音声合成サービスを VOICEVOX と AI Voice で検討中。品質・コスト・レイテンシのバランスが重要。

  **選択肢**:
  - **A**: VOICEVOX（ローカル、無料、品質 ◎）
  - **B**: AI Voice（クラウド、高品質、有料）

  **推奨方式**: A — ローカル実行でレイテンシ低く、無料で継続利用しやすい。

選択肢:
  1. この内容で追記する
  2. 修正する（どこを直すか入力）
  3. 選択肢を追加する
  4. キャンセル
  5. その他（入力してください）

補足: サブ Q を含める場合は ISSUE-XXX-1 のサブ番号で展開する
```
