# ステップ構造テンプレート

対話形式スキルのステップは以下の形式で記述する：

```markdown
## {フェーズ名 or このフェーズに入る条件}
### ステップ({ステップタイトル})
条件: {このステップが実行される条件}
入力: {ユーザーから必要な入力。未提供の場合は入力を求める}

AI実施タスク:
  - タスクA: {概要}
    - サブエージェント: {実行方式}
    - 必読情報: {渡すスキル・ファイルパス・コンテキスト}
  - タスクB: {概要}

出力: {ユーザーに提示する内容}
選択肢:
  1. 選択肢1
  2. 選択肢2
  3. その他（入力してください）
補足: {追加情報}
```

## 各フィールドの意味

| フィールド       | 書くべき内容                                                                                                       |
| ---------------- | ------------------------------------------------------------------------------------------------------------------ |
| **条件**         | どの状態・選択からこのステップに到達するか                                                                         |
| **入力**         | このステップの実行に必要なユーザー入力                                                                             |
| **AI実施タスク** | **出力を生成する前に**AIが裏で行う処理（データ収集、分析、ファイル読み込み、サブエージェント実行、memory記録など） |
| **出力**         | AI実施タスクの結果をもとにユーザーに提示するテキスト                                                               |
| **選択肢**       | ユーザーが次にとれるアクション（番号付き。各選択肢に遷移先ステップを明記）                                         |
| **補足**         | 実装上の注意、参照すべきファイル、例外処理、分岐先の一覧など                                                       |

> **注意: AI実施タスクに「ルーティング（分岐先）」を書かない**
>
> AI実施タスクは「ユーザーに出力を見せる**前に**AIが行う処理」。
> 「ユーザーの選択に応じてどのステップに飛ぶか」は**選択肢**または**補足**に書くこと。
>
> - NG: `AI実施タスク: 新規作成 → ステップ2へ、編集 → ステップ9へ`
> - OK: `AI実施タスク: ワークスペース内の既存ファイルを走査し、参考情報を収集する`

## デフォルト選択肢

SKILL.md上の各ステップの選択肢に明示されていなくても、AIは必ず以下を選択肢に含めること:
- **その他（入力してください）**

## 選択肢の提示形式

選択肢は必ず番号付きで提示し、その下に進捗バーを表示すること:
```
1. 選択肢1
2. 選択肢2
3. その他（入力してください）

[████████░░░░░░░░░░░░] ステップX.Y / 全Nフェーズ
```
