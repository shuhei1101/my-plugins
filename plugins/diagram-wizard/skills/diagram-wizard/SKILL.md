# diagram-wizard — 図生成ウィザード

## このスキルについて

Mermaid図やプレーンテキスト図を対話形式で作成するスキル。
粒度を段階的にコントロールし、人が見やすい図を生成する。

### 対応する図の種類
1. **おまかせ図作成** — 対象を分析し最適な図の種類を自動選択
2. **フロー図** — 処理の流れ・分岐・ループを可視化
3. **ユースケース図** — アクターとシステム機能の関係を可視化（flowchartで代替）
4. **クラス図** — クラス間の継承・依存・集約関係を可視化
5. **シーケンス図** — オブジェクト間のメッセージを時系列で可視化
6. **ER図** — DBエンティティ間の関係を可視化
7. **フォルダ構成図** — プロジェクトのディレクトリ構造を可視化（plaintext）
8. **アスキーアート** — テキストベースの図・装飾を生成
9. **既存図の編集・改善** — 既存のmermaidコードを読み込んで改善

### 主な特徴
- **粒度制御**: 大→中→小の段階的な詳細化（ユーザー自由入力で粒度指定）
- **可読性重視**: 要素数が多い場合はセクション分割を自動提案
- **コメント付き**: mermaidコードに `%%` コメント、`note` アノテーションを付与
- **方向ルール**: クラス図=縦、シーケンス図=横、フロー図=縦
- **連続作成**: 一度のセッションで複数の図を続けて作成可能

## 図の生成ルール（概要）

詳細は `references/mermaid_reference.md` を参照。

### 方向ルール
| 図の種類 | デフォルト方向 | 備考 |
|----------|---------------|------|
| クラス図 | 縦（TB） | クラス数が多い場合は横（LR）も可 |
| シーケンス図 | 横 | mermaidのsequenceDiagramは常に横 |
| フロー図 | 縦（TD） | |
| ユースケース図 | 横（LR） | flowchartで代替 |
| ER図 | - | mermaid自動レイアウト |

### コメント・ノートルール
- mermaidコードには必ず `%%` コメントでセクション区切りと説明を入れる
- 重要な要素には `note` / `note for X` でアノテーションを付与する
- フォルダ構成図は `#` でコメント

### 可読性ルール
- 1枚の図のノード数目安: 15以下（超過時は分割提案）
- mermaidにはズーム機能がないため、要素数が多いと図が小さくなる
- 要素超過時は粒度の段階を増やすか、セクション分割する
