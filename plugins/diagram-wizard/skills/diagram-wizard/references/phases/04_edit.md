# フェーズ5: 既存図の編集

メインメニューで「既存図の編集・改善」が選択された場合、またはステップ4.3から遷移した場合に使用する。

---

## ステップ5.1（既存図の読み込み）

**条件**: メインメニューで「既存図の編集」が選択された、またはステップ4.3から遷移

**入力**: ファイルパス or 直接貼り付けられたMermaid/テキストコード

**AI実施タスク**:
  - 入力方法を確認する（ファイルパス指定 or 直接貼り付け）
  - ファイルパスが指定された場合:
    - ファイルを読み込む
    - Markdownファイルの場合はMermaidコードブロックを抽出する
    - 複数のコードブロックがある場合は一覧表示し、編集対象を選択させる
  - 直接貼り付けの場合:
    - 貼り付けられたコードを受け取る
  - 図の種類を自動判定する（classDiagram, sequenceDiagram, flowchart, erDiagram 等 / プレーンテキスト: フォルダ構成図, アスキーアート）
  - 図の構造を解析する（要素数、関係性の数、方向、コメントの有無、粒度）
  - 解析結果のサマリーを表示する

**出力**:
  - 読み込んだ図の表示（コードブロック）
  - 解析結果サマリー:
    - 図の種類
    - 要素数
    - 関係性の数
    - 方向設定
    - コメントの有無・充実度
    - 検出された問題点（あれば）

**選択肢**:
  1. 改善提案を見る → ステップ5.2へ
  2. 直接編集内容を指定する → ステップ5.3へ
  3. 別のファイル/コードを読み込む → ステップ5.1を再実行
  4. その他（入力してください）

**補足**:
  - 対応するファイル形式: `.md`, `.mmd`, `.mermaid`, `.txt`, プレーンテキスト直接貼り付け
  - ファイルが見つからない場合はエラーメッセージとともにパスの再入力を促す
  - Mermaid構文エラーがある場合は、エラー箇所を特定して報告する

---

## ステップ5.2（改善提案）

**条件**: ステップ5.1で図の読み込み・解析が完了

**入力**: 解析済みの既存図

**AI実施タスク**:
  - 以下の観点で図を分析し、改善点を洗い出す:
    - **可読性**: ノード名・ラベルの分かりやすさ、略語の多用、長すぎるラベル
    - **コメント**: `%%` コメントの有無・充実度。セクション区切りのコメントが不足していないか
    - **構造**: グループ化の適切さ、関連する要素が離れていないか、レイヤー分けが適切か
    - **要素数**: 1つの図に要素が多すぎないか（目安: 15要素超で警告）。多い場合は分割を提案
    - **方向の最適化**: 現在の方向（TD/LR等）が図の種類・内容に適しているか
    - **スタイル**: 色分け・線種の活用が適切か、視覚的な強弱がついているか
    - **アノテーション**: `note` の活用が不足していないか、重要な補足情報が欠けていないか
  - 各改善点に優先度（高/中/低）を付与する
  - 改善前→改善後のイメージを具体的に示す

**出力**:
  - 改善提案一覧を表形式で提示:

  | No. | 観点 | 優先度 | 現状の問題 | 改善内容 |
  | --- | ---- | ------ | ---------- | -------- |
  | 1   | コメント | 高 | セクションコメントなし | セクションごとに `%%` コメントを追加 |
  | 2   | 要素数 | 高 | 20要素で過密 | サブシステム単位で3図に分割 |
  | 3   | 方向 | 中 | TD設定だが横長の図 | LRに変更して可読性向上 |

**選択肢**:
  1. すべての改善を適用する → ステップ5.3へ
  2. 適用する改善を選択する（番号をカンマ区切りで指定）→ ステップ5.3へ
  3. 改善提案は不要、直接編集内容を指定する → ステップ5.3へ
  4. その他（入力してください）

**補足**:
  - 改善提案はあくまで提案であり、ユーザーの判断を優先する
  - 要素数が多すぎる場合の分割提案では、具体的な分割案（どの要素をどの図に振り分けるか）も提示する
  - 図の種類を変更した方が適切な場合もその旨を提案する

---

## ステップ5.3（編集・確認）

**条件**: ステップ5.2で改善内容が確定、またはステップ5.1からユーザーが直接編集を指定

**入力**: 既存の図、適用する改善内容 or ユーザー指定の編集内容

**AI実施タスク**:
  - 選択された改善内容またはユーザー指定の編集を適用して図を再生成する
  - 編集前と編集後の比較を表示する:
    - **変更点サマリー**（箇条書き）
    - **Before**: 編集前のコード（コードブロック）
    - **After**: 編集後のコード（コードブロック）
  - 分割が行われた場合は、分割後の各図を個別に表示する

**出力**:
  - 変更点サマリー（箇条書き）
  - Before/After比較（両方のコードブロック）
  - 「**編集後の図を確認してください。**」

**選択肢**:
  1. 編集内容でOK、出力に進む → ステップ4.1へ
  2. さらに修正したい → ステップ5.3を再実行（追加の修正内容を指定）
  3. 編集前に戻す → ステップ5.1へ（元の図を再読み込み）
  4. 別の改善も適用する → ステップ5.2へ戻る
  5. その他（入力してください）

**補足**:
  - Before/After比較は、変更が小さい場合でも必ず両方表示する
  - 大幅な変更の場合は、段階的に適用する選択肢も提示する
  - 編集ループは何度でも繰り返し可能
  - 出力フェーズ（ステップ4.1）に進む際は、編集後の図を出力対象とする
