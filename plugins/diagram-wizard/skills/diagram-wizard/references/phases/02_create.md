# フェーズ3: 作成

---

## ステップ3.1（図の作成）

**条件**: フェーズ2で図の構造・粒度・方向が確定済み

**入力**: 確定済みの図の構成（要素一覧、関係性、粒度レベル、方向設定）、図の種類

**AI実施タスク**:
  - 確定した構成情報をもとに図を生成する
  - 図の種類に応じた生成ルールを適用する:
    - **Mermaid共通ルール**:
      - コードの先頭にタイトル・概要の `%%` コメントを記載する
      - セクションごとに `%%` コメントで区切り、可読性・保守性を確保する
      - 適切な箇所に `note` / `note for X` アノテーションを付与する（補足説明・注意点・仕様メモなど）
    - **クラス図**: デフォルトは縦方向（TD/TB）。クラス数が多い場合（目安: 横に4つ以上並ぶ場合）は横方向（LR）に切り替える
    - **シーケンス図**: 横方向が自然（mermaidのsequenceDiagramは常に横方向）
    - **フロー図**: デフォルトは縦方向（TD/TB）
    - **ユースケース図**: Mermaidのflowchart（LR）で模倣。アクターはスタジアム型 `([アクター名])`、ユースケースは円形 `((ユースケース名))` で表現
    - **ER図**: Mermaidの `erDiagram` 構文を使用。エンティティ名・属性名は英語、コメントで日本語補足
    - **フォルダ構成図**: プレーンテキストでツリー表記。コメントは `#` で記載。重要でないファイルは `...` で省略（AIが重要度を自動判断: 設定ファイル・エントリポイント・主要モジュールは残す。生成物・キャッシュ・ボイラープレートは省略対象）
    - **アスキーアート**: プレーンテキストで描画。罫線文字（`─`, `│`, `┌`, `┐`, `└`, `┘`, `├`, `┤`, `┬`, `┴`, `┼`）を活用
  - 複数粒度レベルが計画されている場合は、最も粗い粒度（概要レベル）から生成を開始する
  - 生成したコードをコードブロックで表示する

**出力**:
  - 生成した図（Mermaidコードブロック or プレーンテキスト）
  - 図の概要説明（どの範囲を表現しているか、粒度レベルの説明）
  - 確認の促し: 「作成した図を確認してください。Mermaidの場合はプレビューツール（Mermaid Live Editor等）での表示確認をおすすめします。」

**選択肢**:
  1. この図でOK → ステップ3.2へ
  2. 修正したい箇所がある（具体的に指定してください）→ 修正を反映してステップ3.1を再実行
  3. 要素を追加・削除したい → 修正を反映してステップ3.1を再実行
  4. 方向（縦/横）を変更したい → 方向を変更してステップ3.1を再実行
  5. 最初から作り直す → ステップ2.1へ戻る
  6. その他（入力してください）

**補足**:
  - ユーザーが修正を要求した場合は、変更箇所のみを差分で説明してからコード全体を再表示する
  - Mermaid構文エラーが発生しやすい箇所（特殊文字、長いラベル、ネスト構造）には注意を払い、エスケープ処理を適切に行う
  - フォルダ構成図で省略判断に迷う場合は、ユーザーに確認する
  - 図の生成ルールの詳細は `references/mermaid_reference.md` を参照すること

## ステップ3.2（図の確認）

**条件**: ステップ3.1で図が生成済み、ユーザーが「この図でOK」を選択

**入力**: 生成済みの図、ユーザーのフィードバック

**AI実施タスク**:
  - ユーザーに図の最終確認を促す
  - 図の品質チェックを実施する:
    - 要素の過不足がないか
    - ラベル・命名が適切か
    - コメント（`%%`）が十分に付与されているか
    - 方向・レイアウトが見やすいか
  - 品質チェック結果に問題があればユーザーに報告する
  - 複数粒度が計画されている場合は、次の粒度レベルがある旨を案内する

**出力**:
  - 「**作成した図を確認してください。**」（最終確認の明示的な促し）
  - 品質チェック結果（問題がある場合のみ表示）
  - 次の粒度レベルがある場合はその旨の案内

**選択肢**:
  1. 確認OK、次の粒度へ深掘りする → ステップ3.3へ（複数粒度が計画されている場合のみ表示）
  2. 確認OK、出力に進む → ステップ4.1へ
  3. やっぱり修正したい → ステップ3.1へ戻る（修正内容を指定）
  4. その他（入力してください）

**補足**:
  - **必ずユーザーの明示的な確認を得てから次に進む**こと。自動的に次のステップに進んではならない
  - ユーザーが修正を指示した場合はステップ3.1に戻り、修正を反映して再生成する

## ステップ3.3（次の粒度へ深掘り）

**条件**: 複数粒度レベルが計画されており、ユーザーが深掘りを選択

**入力**: 現在の粒度レベルの図、次の粒度レベルの計画、ユーザーが深掘りしたいセクション

**AI実施タスク**:
  - 現在の図からセクション一覧を抽出し、どの部分を深掘りするか提案する
  - ユーザーが選択したセクションについて、次の粒度レベルの構造を設計する（ステップ2.1相当のフローを簡易実施）:
    - 対象セクション内の詳細要素を洗い出す
    - 要素間の関係性を整理する
    - 方向・レイアウトを提案する
  - 確認後、ステップ3.1に戻って詳細図を生成する
  - 全粒度レベルが完了したか、ユーザーが満足した時点で出力フェーズへ誘導する

**出力**:
  - 深掘り可能なセクション一覧（番号付き）
  - 選択されたセクションの詳細構造案
  - 残りの粒度レベル数の案内

**選択肢**:
  1. セクションを選択して深掘りする（番号で指定）→ 構造確認後ステップ3.1へ
  2. 複数セクションをまとめて深掘りする → 構造確認後ステップ3.1へ
  3. 深掘りは十分、出力に進む → ステップ4.1へ
  4. その他（入力してください）

**補足**:
  - 深掘り時は親図との整合性を維持する（要素名・関係性の不一致がないように注意）
  - 深掘りのたびに「概要図 → 詳細図」の対応関係をユーザーに示す
  - ユーザーが「もう十分」と判断した場合は、残りの粒度レベルが未完了でも出力フェーズに進む
