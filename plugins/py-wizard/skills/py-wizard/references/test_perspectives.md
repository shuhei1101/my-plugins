# テスト観点テンプレート

ステップ4.1で使用するテスト観点チェックリスト。AIが実装内容に応じて推奨にチェック済みにして提示する。

---

```
[✓] 1. 正常系 — 期待通りの入力で正しく動作するか
[✓] 2. 異常系 — 不正な入力でエラーハンドリングが機能するか
[ ] 3. 境界値 — 最小値・最大値・空入力などの境界条件
[✓] 4. 環境変数パターン — 環境変数の値が変わった場合の動作
[✓] 5. 外部ライブラリモック — 外部ライブラリの返却値パターン差異
[ ] 6. ファイルI/O — 入出力ファイルの存在・権限・フォーマット違い
[ ] 7. 設定値パターン — config/envの組み合わせ（デフォルト・上書き・未設定）
[ ] 8. 連携ツール — 連携先ツールのパス不正・未インストール時の挙動
[ ] 9. 並行処理 — 複数プロセス・スレッドでの実行時の挙動
[ ] 10. パフォーマンス — 大量データ・長時間実行時の挙動
```

---

## 各観点の詳細

### 1. 正常系
- 基本的な使用シナリオで期待通りの結果が得られるか
- 代表的な入力パターンでの動作確認

### 2. 異常系
- 不正な型・フォーマットの入力
- 必須パラメータの欠落
- 想定外の値（null, 空文字, 極端に大きい値）

### 3. 境界値
- 最小値・最大値
- 空入力・空リスト
- 文字数上限・下限

### 4. 環境変数パターン
- 環境変数が設定されている場合の動作
- 環境変数が未設定（デフォルト値フォールバック）
- 環境変数が不正な値の場合
- `tests/mocks/mock_env.py` のパターンを使用

### 5. 外部ライブラリモック
- 正常レスポンスのパターン
- エラーレスポンス（タイムアウト、認証エラー等）
- 空レスポンス
- `tests/mocks/mock_externals.py` のパターンを使用

### 6. ファイルI/O
- ファイルが存在しない場合
- ファイル権限がない場合
- フォーマットが不正な場合
- 空ファイル

### 7. 設定値パターン
- config.pyのデフォルト値のみ
- .envで上書きした場合
- 引数で上書きした場合
- 設定値が未設定の場合

### 8. 連携ツール
- 連携先ツールのパスが不正
- 連携先ツールが未インストール
- 連携先ツールのバージョン不一致

### 9. 並行処理
- 複数プロセスでの同時実行
- ファイルロック競合
- リソース競合

### 10. パフォーマンス
- 大量データ処理時の処理時間
- メモリ使用量
- 長時間実行時の安定性
