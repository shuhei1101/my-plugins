# README作成ガイド

## セクション構成

READMEは以下の順序で作成する:

**重要**: ユーザーが最初に必要とする情報（インストール・使い方）を上部に配置すること

1. 概要
2. インストール方法
3. 使い方
   - GUI起動方法
   - CLI起動方法
   - 設定方法
4. ツール仕様
5. 開発技術
6. フォルダ構成
7. ライセンス（MIT）

---

## 出力形式ルール

- **箇条書き+インデント形式**で統一する
- シンプル優先、決まった書き方はない
- 表形式は使わない（フォルダ構成のツリー表示は例外）

### 形式例

```markdown
## 開発技術

- 言語: Python3.11~
- 使用ライブラリ:
  - GUI: tkinter
  - 音声変換: ffmpeg
  - 文字起こし: faster-whisper
- 外部ツール:
  - 動画変換: ffmpeg
```

## セクション別ガイドライン

### 概要
- プロジェクトの目的を1〜3行で簡潔に記述
- 「何をするツールか」が一目でわかるように

### インストール方法
- **Windowsユーザ向け**（Pythonインストールから記載）
- 自動セットアップ（推奨）と手動セットアップの2方法を提示
- setup/フォルダのセットアップ用batの使い方を含める
- セットアップ確認手順を記載

### 使い方
- GUI起動方法（bat実行）
- CLI起動方法（引数指定）
- モード別の使い方
- 設定方法（.envファイル編集、GUI設定画面）
- 連携先ツールのパス設定（該当する場合）

### ツール仕様
- セクション冒頭にツール構成フロー図（Mermaid）を配置する
  - **注意**: Mermaid図内で改行する場合は `\n` ではなく `<br>` を使用すること
- 以下の固定カテゴリで整理:
  - **機能一覧**: 主要機能を箇条書き
  - **入出力**: 入力形式・出力形式・対応フォーマット等
  - **パフォーマンス**: 処理速度・並列処理・リソース管理等
  - **インターフェース**: GUI/CLI/API等の利用方法
  - **設定**: 設定項目・環境変数・デフォルト値等
- サブ項目はインデントで表現

### 開発技術
- 使用ライブラリ・外部ツール・APIを用途付きで列挙
- **作成時にMCP（context7）やWeb検索で最新情報を調査する**
- **技術選定プロセス**:
  - ユーザー指定技術には内部管理として `[ユーザ指定]` ラベルを付与（確認時のみ）
  - AI推奨技術には内部管理として `[推奨]` ラベルを付与（確認時のみ）
  - 最終READMEにはラベルを含めず、用途と技術名のみ記載

### フォルダ構成
- ツリー形式で表示
- 各フォルダ・ファイルに簡潔な説明をコメントで付加

### ライセンス
- MIT固定
