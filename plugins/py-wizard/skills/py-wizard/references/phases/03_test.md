# フェーズ4: テスト

---

## ステップ4.1（テスト観点選択）

**条件**: ステップ3.3でコード実装が完了した場合

**入力**: -

**AI実施タスク**:
- 実装内容に対して適用可能なテスト観点を判定する
- `references/test_perspectives.md` の10個のテンプレートからチェックリスト形式で提示する
- AIが実装内容に応じて推奨にチェック済みにする

**出力**:
- テスト観点チェックリスト（`references/test_perspectives.md` 参照）
- 「テスト観点を選択してください（番号指定、複数可）」

**選択肢**:
1. この選択で決定する（編集後のリストを貼り付けてもOK）
2. すべて採用する
3. 修正内容を伝える
4. その他（入力してください）

**補足**:
- 統合テストのみ作成（単体テストは不要）
- 外部API・外部ライブラリの呼び出しはモック化

---

## ステップ4.2（モックファイル設計）

**条件**: ステップ4.1でテスト観点が選択された場合

**入力**: -

**AI実施タスク**:
- 選択されたテスト観点に基づき、必要なモックファイルを設計する
- モックファイルは再利用可能な別ファイルとして作成（毎回作成しない）:
  - `tests/mocks/mock_env.py` — 環境変数モック（パターン別に定義）
  - `tests/mocks/mock_externals.py` — 外部ライブラリ返却値モック（パターン別に定義）
  - 必要に応じて追加モックファイル
- 各モックにパターンを定義（正常値、異常値、境界値など）

**出力**:
- モックファイル構成の一覧
- 各モックファイルに含めるパターンの概要

**選択肢**:
1. OK → テストコード実装へ
2. 修正内容を伝える → 修正して再提示
3. その他（入力してください）

**補足**:
- テストコードでは `from tests.mocks.mock_env import ...` のようにインポートして使い分ける
- モックパターンの追加は後からでも可能

---

## ステップ4.3（テストコード実装）

**条件**: ステップ4.2でモックファイル設計が完了した場合

**入力**: -

**AI実施タスク**:
- 選択されたテスト観点に基づいて統合テストコードを実装する
- モックファイルからインポートして使い分ける
- テストコードは `tests/` フォルダに配置（機能ミラー構成）
  ```
  tests/
  ├── mocks/
  │   ├── mock_env.py
  │   └── mock_externals.py
  ├── conftest.py
  ├── {機能名}/
  │   ├── conftest.py
  │   └── test_{機能名}.py
  └── ...
  ```
- pytest形式で記述
- テスト関数名・コメントは英語

**出力**:
- 実装したテストファイル一覧
- 各テストファイルのテストケース概要

**選択肢**:
1. OK → テスト実行へ
2. テスト内容を確認する（ファイル名を指定）
3. 修正内容を伝える → 修正して再実装
4. その他（入力してください）

---

## ステップ4.4（テスト実行・結果確認）

**条件**: ステップ4.3でテストコード実装が完了した場合

**入力**: -

**AI実施タスク**:
- `pytest` コマンドを実行する
- 実行結果を整理して表示する
- 失敗テストがある場合:
  - 失敗箇所・エラー内容を整理
  - 原因の仮説を提示
  - 修正案を提案

**出力**:
- テスト実行結果サマリー（pass/fail数）
- 失敗テストがある場合: 失敗箇所・エラー内容・原因仮説・修正案

**選択肢**:
- 全テストpass時:
  1. OK → プロジェクト専用スキル・ドキュメント更新へ
  2. テストを追加したい → ステップ4.1へ戻る
  3. その他（入力してください）
- 失敗テストあり時:
  1. 修正案で修正する → コード修正後、再度テスト実行（このステップをループ）
  2. 一緒に原因を考える → 対話で原因分析
  3. 失敗テストをスキップして進む
  4. その他（入力してください）

---

## ステップ4.5（プロジェクト専用スキル・ドキュメント更新）

**条件**: ステップ4.4でテストがpassした場合

**入力**: -

**AI実施タスク**:
- プロジェクト専用スキルに新規・変更ファイルを反映する（ファイル名・概要・パス）
- README.mdの該当セクションを更新（機能一覧・フォルダ構成など、変更があれば）
- テスト関連ファイルもプロジェクト専用スキルに追加

**出力**:
- 更新した内容の差分サマリー

**選択肢**:
1. OK → 完了フェーズへ
2. 修正内容を伝える → 修正して再更新
3. 追加の実装がある → ステップ3.1へ戻る
4. その他（入力してください）
