# フェーズ3: 実装・改修

---

## ステップ3.1（要件入力）

**条件**: 実装・改修モードが選択された場合、または新規作成から移行した場合

**入力**: ユーザーが要件を自由入力

**AI実施タスク**:
- プロジェクト専用スキルが存在する場合は読み込む
- カレントディレクトリのコードを自動読み取り（プロジェクト専用スキルがない場合）
- ユーザーが指定したファイルがあればそちらも読み込む
- 要件の内容を整理する

**出力**:
- 「実装・改修したい内容を教えてください」
- プロジェクト専用スキルがある場合は現在の構造を表示
- なければ「対象プロジェクトのパスを指定してください」

**選択肢**: なし（自由入力で要件を伝えてもらう）

**補足**:
- プロジェクト専用スキルがない場合、ステップ3.2の後にスキル生成を提案する
- 要件入力後、ステップ3.2（実装計画まとめ）へ進む

---

## ステップ3.2（実装計画まとめ）

**条件**: ステップ3.1で要件が入力された場合

**入力**: -

**AI実施タスク**:
- 要件をもとに実装計画を整理する:
  - 新規作成するモジュール（ファイル名・配置先・概要）
  - 修正するモジュール（ファイル名・変更内容）
  - クラス設計（抽象クラス・インターフェースの活用箇所）
  - 連携先ツールのパス設定が必要な場合はその旨も記載
- **MCP（context7）やWeb検索を使って技術的な最新情報を調査する**（必要に応じて）

**出力**:
- 実装計画の一覧（箇条書き+インデント形式）
- 「この計画で実装に進みますか？」

**選択肢**:
1. OK → コード実装へ
2. 修正内容を伝える → 修正して再提示
3. 要件を追加・変更する → ステップ3.1へ戻る
4. その他（入力してください）

**補足**:
- 低レベル処理は抽象クラス・インターフェースで設計し、再利用性・テスト容易性を確保
- print文・ログ文はすべて英語で記述する旨を計画に含める

## ステップ3.3（コード実装）

**条件**: ステップ3.2で実装計画が承認された場合

**入力**: -

**AI実施タスク**:
- 実装計画に基づいてコードを実装する
- 実装ルール（`references/python_rules.md` に従う）:
  - typing厳格（Literal, Union等を積極的に使用）
  - docstringはreStructuredText形式
  - 複雑な箇所には適宜コメント
  - リーダブルコード意識（仮変数の活用）
  - デザインパターン適用（テンプレート、ストラテジー等、適材適所）
  - main.pyは高レベルのみ、低レベル処理は抽象クラス・インターフェースへ
  - loggerをがっつり入れる（デバッグ追跡用）
  - **ログ出力の必須要件**: `logger.py` の `setup_logger()` は `StreamHandler` + `FileHandler(log/YYYYMMDD_HHMMSS_{package_name}.log)` を両方アタッチし（実行ごとにタイムスタンプ付き新規ファイル、過去実行を上書きしない）、`LOG_DIR` は `log/` フォルダを自動作成する。`main.py` / `__main__.py` の起動直後で必ず `setup_logger()` を呼ぶ（参考: `voice-paste/voice_paste/logger.py`）
  - **.bat ランチャーの必須要件**: `run.bat` / `bat/*.bat` は stdout/stderr を `log/<タイムスタンプ>_run_bat.log`（実行ごとにタイムスタンプ付き新規ファイル、PowerShell `Get-Date` で生成。bat ログと python ログが同じ粒度のタイムスタンプを持つことで同一実行分をソート順で相関可能）に追記リダイレクトし、venv自動有効化・非ゼロ終了時 `pause` を組み込む（参考: `voice-paste/run.bat`、正準スニペットおよび長時間コマンド向けコンソール／ログ同時出力パターン（PowerShell `Write-Host` + `Add-Content -Encoding utf8`、CP932 環境での文字化け対策込み）は `references/python_rules.md` の「.bat ランチャーログ」節）
  - **【ASCII限定】`.bat` ファイルの中身は ASCII 文字のみ**（日本語コメント・echo文字列禁止）。`cmd.exe` は bat をシステムANSIコードページ（CP932）でパースするため、UTF-8保存の日本語バイトが CP932 リードバイトとして誤認され、続くコマンドが壊れる（例: `'etlocal' is not recognized`）。日本語説明は `README.md` に書く
  - print文・ログ文はすべて英語
  - 設定値は引数で受け取れる設計（優先順位: 環境変数 → 引数上書き）

**出力**:
- 実装したファイル一覧
- 各ファイルの概要

**選択肢**:
1. OK → テストフェーズへ
2. 実装内容を確認する（ファイル名を指定）
3. 修正内容を伝える → 修正して再実装
4. その他（入力してください）
