# フェーズE: 会話モード

---

## ステップ13(1対1会話)

**条件**: モード選択で「エージェントと会話する」が選択された場合、またはステップ8（ファイル出力）後に動作確認として選択された場合

**入力**: エージェント名またはパス（未指定の場合は一覧から選択を促す）

**AI実施タスク**:
- エージェントが未指定の場合: 全スコープからエージェント一覧を表示し、選択を促す
- 指定されたエージェントの `.md` ファイルを読み込む
- エージェントのペルソナ（役割・コアバリュー・推論スタイル・回答形式）をコンテキストに設定する
- 会話モードに入る:
  - ユーザーの入力をエージェントのペルソナで応答する
  - ユーザーがタスク実行を求めた場合（例: 「このコード修正して」）は、サブエージェントを起動してエージェントのペルソナでタスクを実行させる
- 会話ログを `~/.claude/skill-memory/agent-wizard/{YYYYMMDDHHMMSS}_chat_{agent-name}.md` に記録する

**出力**:
- 「🤖 {エージェント名} として会話を開始します。終了するときは「終了」と入力してください。」
- エージェントのペルソナに基づいた応答

**選択肢**: なし（会話モード中は自由入力。以下のコマンドを受け付ける）
- 「終了」→ 会話モードを抜けてステップ1（モード選択）へ
- 「タスク: {指示}」→ サブエージェントを起動してエージェントのペルソナでタスク実行

**補足**:
- 会話モード中はスキルのステップフローから一時的に離脱する。ユーザーが「終了」と言うまで継続
- タスク実行時のサブエージェントには、エージェントファイルの内容をシステムプロンプトとして渡し、toolsフィールドで指定されたツールを利用可能にする
- ステップ8から来た場合は、作ったばかりのエージェントの動作確認ができるため積極的に提案する

## ステップ14(エージェント間会話)

**条件**: モード選択で「エージェント同士で会話させる」が選択された場合

**入力**: -

**AI実施タスク**:
- 全スコープからエージェント一覧を取得して表示する
- ユーザーに以下を指定してもらう:
  - 参加エージェント（2体以上）
  - 会話トピック
  - ラウンド数（デフォルト: 3往復）
- 指定されたエージェントの `.md` ファイルを読み込み、各ペルソナを把握する
- 会話ログファイル `~/.claude/skill-memory/agent-wizard/{YYYYMMDDHHMMSS}_debate_{topic-slug}.md` を作成
- サブエージェント順次起動方式で会話を実行する:
  1. エージェントAをサブエージェントとして起動し、トピックについて発言させる
  2. Aの発言をエージェントBのプロンプトに含めてサブエージェントとして起動し、応答させる
  3. Bの発言をAに渡して応答させる
  4. 指定ラウンド数に達するまでループ
  5. 3体以上の場合はラウンドロビンで順番に発言
- 各発言をリアルタイムでユーザーに表示しつつ、ログファイルに追記していく
- 全ラウンド終了後、会話のサマリーを生成する

**出力**:
- 「以下の設定でエージェント間会話を開始します」
  - 参加者: {エージェント名一覧}
  - トピック: {トピック}
  - ラウンド: {N}往復
- 会話ログ（リアルタイム表示）:
  ```
  🔵 [cost-conscious] コストの観点から言えば...
  🟢 [quality-first] 品質を考慮すると...
  🔵 [cost-conscious] それは理解できるが...
  ```
- 完了後: サマリー + ログファイルパス

**選択肢**:
  1. 会話ログを確認する
  2. 同じエージェントで別のトピックを議論させる
  3. エージェントを変えてもう一度
  4. 戻る → ステップ1（モード選択）へ
  5. その他（入力してください）

**補足**:
- 各サブエージェントには「他のエージェントの発言に対して、自分のペルソナの観点から具体的に反応すること」と指示する。一般論の応酬にならないようにする
- Agent Teams が将来安定したら切り替えオプションを追加する
