# フェーズB: 新規作成フロー

---

## ステップ2(スコープ・配置先確認)

**条件**: モード選択で「エージェントを新規作成する」が選択された場合、またはリクエストから新規作成意図が検出された場合

**入力**: ユーザーのリクエスト（配置先が指定されていれば抽出する）

**AI実施タスク**:
- 3つのスコープの既存 `agents/` フォルダを走査する:
  - ユーザーレベル: `~/.claude/agents/`
  - プロジェクトレベル: `.claude/agents/`
  - プラグイン内: 現在のワークスペース内のプラグインの `agents/` を検出
- 各スコープのエージェント数を集計する
- 配置先がリクエストで既に指定されている場合（例:「ideaにエージェント追加して」）、パスを自動特定してこのステップの選択をスキップ可能にする
- `~/.claude/skill-memory/agent-wizard/{YYYYMMDDHHMMSS}_agent_session.md` を更新

**出力**:
- 各スコープの状態（フォルダ有無・既存エージェント数）をテーブルで表示:

  | スコープ | パス | 状態 | エージェント数 |
  |---|---|---|---|
  | ユーザー | `~/.claude/agents/` | ✓ 存在 | 5 |
  | プロジェクト | `.claude/agents/` | — なし | 0 |
  | プラグイン | `plugins/idea/skills/idea/agents/` | ✓ 存在 | 23 |

- 「どこに作成しますか？」

**選択肢**:
  1. ユーザーレベル（`~/.claude/agents/`）— 全プロジェクトで使える
  2. プロジェクトレベル（`.claude/agents/`）— このプロジェクト専用
  3. プラグイン内（プラグイン名を指定）— プラグインとして配布可能
  4. パスを直接指定する
  5. その他（入力してください）

**補足**:
- フォルダが存在しない場合は、ステップ8（ファイル出力）の時点で自動作成する
- プラグイン内を選択した場合は、対象プラグインのスキルパスを特定して `agents/` 配下に配置する

---

## ステップ3(役割・ペルソナ設計)

**条件**: スコープ・配置先が確定した後

**入力**: ユーザーが伝えるエージェントの目的・役割（未提供の場合は入力を促す）

**AI実施タスク**:
- ユーザーの入力からエージェントの方向性を整理する:
  - **エージェントのタイプ判定**: ペルソナ型（意見生成） / 分析型（採点・評価） / 調査型（外部情報参照） / タスク実行型（ファイル操作等）
  - タイプに応じて `/idea` の既存エージェントや `references/agent_file_spec.md` のパターンを参照する
- `/idea` を使って、エージェント設計に必要な質問を複数の観点から生成する:
  - 「このエージェントの設計について」
  - 観点: ユーザー体験・実用性・差別化
- 以下の項目をヒアリング or 起案する:
  - **役割**: 1文でこのエージェントが何者かを定義
  - **コアバリュー**: 3〜5個の優先事項
  - **推論スタイル**: どのように考え、どんな視点で回答するか
  - **典型的な懸念/質問例**: このエージェントが気にすること
- `~/.claude/skill-memory/agent-wizard/{YYYYMMDDHHMMSS}_agent_session.md` を更新（エージェント設計案を記録）

**出力**:
- エージェントのタイプ判定結果
- 役割・コアバリュー・推論スタイルのドラフト
- 不明点や確認したいことがあれば質問

**選択肢**:
  1. この内容でOK → ステップ4（フロントマター設定）へ
  2. 修正内容を伝える → 修正して再提示
  3. `/idea` で別の観点からも案を出す
  4. 既存エージェントを参考にしたい（エージェント名やパスを指定）
  5. その他（入力してください）

**補足**:
- ユーザーの説明が曖昧な場合は、いくつか具体例を出して「こういう感じですか？」と確認する
- ペルソナ型の場合は `/idea` の既存エージェント（`agents/implementation/cost-conscious.md` 等）を読み込んで構造の参考にする
- 調査型の場合は `agents/ai-dev/openai-latest.md` 等を参考にする

---

## ステップ4(フロントマター設定)

**条件**: 役割・ペルソナ設計が確定した後

**入力**: ステップ3で確定した役割・コアバリュー・推論スタイル

**AI実施タスク**:
- ステップ3の設計内容からフロントマターの各フィールドを起案する:
  - **name**: 役割からkebab-caseのスラッグを生成（候補を2〜3個提示）
  - **description**: 役割+呼び出し文脈を1文にまとめる。「いつ使うか」を明確にする
  - **model**: エージェントタイプに応じて推奨を提示
    - ペルソナ型 → `haiku`（軽量・高速）
    - 分析・調査型 → `sonnet`（高精度）
    - 複雑な設計 → `opus`
  - **tools**: タイプに応じてデフォルトセットを提案
    - ペルソナ型 → なし（意見生成のみ）
    - 調査型 → `WebFetch`, `WebSearch`
    - タスク実行型 → `Bash`, `Read`, `Edit`, `Glob`, `Grep`
  - **effort / maxTurns / color**: 必要に応じて提案（デフォルトは省略）
- `references/agent_file_spec.md` を参照して全フィールドを把握
- 全フィールドの一覧を表形式で表示し、ユーザーに確認する

**出力**:
- フロントマター案をテーブルで表示:

  | フィールド | 値 | 備考 |
  |---|---|---|
  | name | `{slug}` | |
  | description | `{1文の説明}` | |
  | model | `haiku` | ペルソナ型のため軽量モデル推奨 |
  | tools | — | 意見生成のみのため不要 |
  | effort | — | デフォルト（省略） |
  | maxTurns | — | デフォルト（省略） |
  | color | — | デフォルト（省略） |

- 「この設定でよろしいですか？」

**選択肢**:
  1. この内容でOK → ステップ5（関連リンク収集）へ
  2. フィールドを変更する（フィールド名と値を指定）
  3. toolsを追加/変更する
  4. model を変更する（haiku / sonnet / opus）
  5. その他（入力してください）

**補足**:
- description はトリガー精度に直結する最重要フィールド。曖昧すぎると呼ばれない、広すぎると誤爆する
- `permissionMode`, `disallowedTools`, `skills` は高度なオプション。ユーザーが言及した場合のみ提示する

## ステップ5(関連リンク収集)

**条件**: フロントマター設定が確定した後。ただしペルソナ型（意見生成のみ）で外部情報参照が不要な場合はスキップ可能

**入力**: ステップ3で判定したエージェントタイプ

**AI実施タスク**:
- エージェントタイプが調査型かどうかを判定する
  - 調査型の場合: 「関連リンク一覧セクションが必要です」と案内し、リンク収集を開始
  - ペルソナ型/タスク実行型の場合: 「関連リンクは不要そうですがどうしますか？」と確認し、スキップ選択肢を提示
- 調査型の場合、エージェントの役割に関連しそうなURLカテゴリを提案する:
  - 例: 「公式ドキュメント」「APIリファレンス」「料金ページ」「リリースノート」「GitHub」等
- ユーザーが提供したURLを整理し、Markdown形式で一覧化する
- URLが正しいか WebFetch で簡易チェックする（タイトル取得）

**出力**:
- 調査型の場合:
  - 「以下のカテゴリでリンクを整理します。URLを教えてください」
  - 提案カテゴリ一覧
- ペルソナ型の場合:
  - 「このエージェントは意見生成型なので関連リンクは不要そうですが、追加しますか？」

**選択肢**:
  1. リンクを入力する（URLを貼り付けてください）
  2. リンク不要、スキップする → ステップ6（回答形式設計）へ
  3. AIに関連リンクを検索してもらう（トピックを指定）
  4. その他（入力してください）

**補足**:
- 調査型エージェントではスキップを非推奨とする（関連リンク一覧セクション必須ルール）
- リンクは追加・削除しながら何度でも編集可能。確定したらステップ6へ進む
- リンク収集が複数回に分かれる場合は、都度 `~/.claude/skill-memory/agent-wizard/` に中間結果を記録する

---

## ステップ6(回答形式設計)

**条件**: 関連リンク収集が完了またはスキップされた後

**入力**: ステップ3の役割・コアバリュー・エージェントタイプ

**AI実施タスク**:
- エージェントタイプに応じた回答形式のテンプレートを提案する:
  - **ペルソナ型**: 意見（1〜2文）+ 根拠（2〜3文）
  - **分析型**: 評価結果 + 証拠 + pass/fail
  - **調査型**: 最新情報（WebFetch結果反映）+ 推奨アクション + 参照URL
  - **タスク実行型**: 実行結果 + 変更内容サマリー
  - **命名型**: 候補一覧（3〜5個）+ 各候補のポイント
- `/idea` の既存エージェントから同タイプの回答形式を参照し、具体例を提示する
- ユーザーの要望があればカスタマイズする（テーブル形式、箇条書き、コードブロック等）

**出力**:
- 回答形式のドラフト:
  ```
  ## 回答形式

  以下を提供してください:
  1. {項目1}
  2. {項目2}
  3. {項目3}

  {追加の指示があれば}
  ```
- 「この回答形式でよろしいですか？」

**選択肢**:
  1. この内容でOK → ステップ7（確認・プレビュー）へ
  2. 修正内容を伝える → 修正して再提示
  3. 既存エージェントの回答形式を見たい（エージェント名を指定）
  4. その他（入力してください）

**補足**:
- 回答形式はエージェントの出力品質に大きく影響する。具体的であるほど一貫した回答が得られる
- 「一般論を避け、具体的かつ実践的に」のような品質指示も回答形式セクションに含める

---

## ステップ7(確認・プレビュー)

**条件**: 回答形式設計が確定した後

**入力**: ステップ3〜6で確定した全設計内容

**AI実施タスク**:
- ステップ3〜6の成果物を結合して、完成形のエージェントファイルを組み立てる:
  - フロントマター（ステップ4）
  - タイトル + 役割説明（ステップ3）
  - 関連リンク一覧（ステップ5、あれば）
  - コアバリュー（ステップ3）
  - 推論スタイル（ステップ3）
  - 典型的な懸念/質問例（ステップ3）
  - 回答形式（ステップ6）
- 出力先の暫定パスを組み立てる（ステップ2のスコープ + スラッグ名。カテゴリフォルダはステップ8で確定するため、ここでは暫定表示）
- ファイル全体をコードブロックでプレビュー表示する
- 明らかな問題（descriptionが曖昧、modelとtoolsの不整合等）があればチェックして指摘する

**出力**:
- エージェントファイルの全文プレビュー（コードブロック）
- チェック結果（問題があれば）:
  - description: OK / 改善提案あり
  - model と tools の整合性: OK / 不整合
  - 関連リンク: あり / なし（調査型なのに不足していないか）
- 出力予定パス: `{スコープパス}/agents/{slug}.md`

**選択肢**:
  1. この内容でOK → ステップ8（ファイル出力）へ
  2. 修正箇所を伝える → 該当ステップに戻って修正
  3. 全体を最初から作り直す → ステップ3へ
  4. その他（入力してください）

**補足**:
- ここがファイル出力前の最終確認ポイント。ユーザーが承認するまでファイル書き込みは行わない
- プレビューが長い場合は、フロントマター部分とボディ部分を分けて提示しても良い

---

## ステップ8(ファイル出力)

**条件**: プレビューでユーザーが承認した後

**入力**: ステップ7で確定したエージェントファイル全文、ステップ2で確定したスコープパス

**AI実施タスク**:
- 配置先の `agents/` フォルダの存在を確認し、なければ作成する
- カテゴリフォルダの判定:
  - 配置先に既存のカテゴリフォルダがある場合: 一覧を表示し、どこに入れるか / 新規フォルダを作るかを確認
  - カテゴリフォルダがない場合: フラット配置するか、フォルダを新規作成するか確認
- エージェントファイル（`.md`）を出力する
- 配置先スキルの SKILL.md にエージェント一覧テーブルがある場合、追記が必要か確認する
- `~/.claude/skill-memory/agent-wizard/` にセッション完了情報を記録

**出力**:
- 「以下のパスにエージェントファイルを出力しました」
- 出力パス: `{スコープパス}/agents/{カテゴリ/}{slug}.md`
- SKILL.md への追記が必要な場合はその提案

**選択肢**:
  1. 完了 → ステップ15（サマリー出力）へ
  2. SKILL.md のエージェント一覧テーブルに追記する
  3. 続けて別のエージェントを作成する → ステップ3へ
  4. このエージェントと会話してみる → ステップ13へ
  5. その他（入力してください）

**補足**:
- ファイル出力後に「このエージェントと会話してみる」を選ぶと、作ったばかりのエージェントの動作確認ができる
- SKILL.md の一覧テーブル追記は `/idea` のテーブル形式（名前/概要/パス）に従う
