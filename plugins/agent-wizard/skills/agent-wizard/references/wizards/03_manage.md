## ステップ11(エージェント一覧表示)

**条件**: モード選択で「エージェント一覧を見る」が選択された場合

**入力**: -

**AI実施タスク**:
- 全スコープの `agents/` を走査する:
  - ユーザーレベル: `~/.claude/agents/`
  - プロジェクトレベル: `.claude/agents/`
  - プラグイン内: ワークスペース内のプラグインの `agents/`
- 各エージェントファイルのフロントマター（name / description / model）を読み取る
- スコープ別・カテゴリフォルダ別にグルーピングする
- エージェント総数を集計する

**出力**:
- スコープ別にセクションを分けてテーブル表示:

  ```
  📁 ユーザーレベル (~/.claude/agents/) — 2体

  | 名前 | 概要 | model | パス |
  |---|---|---|---|
  | my-reviewer | コードレビュー用 | sonnet | ~/.claude/agents/my-reviewer.md |
  | ... | | | |

  📁 プラグイン: idea (agents/) — 23体

  ### implementation/ (4体)
  | 名前 | 概要 | model | パス |
  |---|---|---|---|
  | cost-conscious | 開発・運用コスト最小化 | haiku | agents/implementation/cost-conscious.md |
  | ... | | | |
  ```

- 合計: {N}体

**選択肢**:
  1. エージェントを編集する（番号または名前で指定）→ ステップ10へ
  2. エージェントと会話する（番号または名前で指定）→ ステップ13へ
  3. 新しいエージェントを作成する → ステップ2へ
  4. スコープで絞り込んで再表示する
  5. 戻る → ステップ1（モード選択）へ
  6. その他（入力してください）

**補足**:
- エージェントが存在しないスコープは「エージェントなし」と表示し、作成を提案する
- 一覧から直接編集・会話に遷移できるようにして、操作のステップ数を減らす

## ステップ12(カテゴリフォルダ管理)

**条件**: モード選択で「カテゴリフォルダを管理する」が選択された場合

**入力**: 対象スコープまたはパス（指定されていれば）

**AI実施タスク**:
- 対象の `agents/` フォルダの現在の構造を走査する（未指定の場合はスコープ選択を促す）
- フォルダ構成をツリー形式で可視化する
- 各フォルダ内のエージェント数を集計する
- カテゴリ分けされていないフラットなエージェントがある場合、分類案を提案する:
  - `/idea` の既存カテゴリ（implementation / docs / project / review / naming / ai-dev）を参考にする
  - エージェントの description を読み取り、適切なカテゴリを推定する

**出力**:
- 現在のフォルダ構成（ツリー表示）:
  ```
  agents/
  ├── implementation/ (4体)
  ├── docs/ (3体)
  ├── ai-dev/ (6体)
  ├── orphan-agent-a.md  ← 未分類
  └── orphan-agent-b.md  ← 未分類
  ```
- 未分類エージェントがある場合: 分類提案
- 「何をしますか？」

**選択肢**:
  1. 新しいカテゴリフォルダを作成する（フォルダ名を指定）
  2. エージェントを別フォルダに移動する（エージェント名と移動先を指定）
  3. 提案された分類をまとめて適用する
  4. フォルダをリネームする
  5. 戻る → ステップ1（モード選択）へ
  6. その他（入力してください）

**補足**:
- フォルダ移動時はファイルを移動した後、SKILL.md のエージェント一覧テーブルのパスも更新が必要か確認する
- 操作は1つずつ実行→結果表示→次の操作、のループで行う。一括操作は選択肢3のみ
