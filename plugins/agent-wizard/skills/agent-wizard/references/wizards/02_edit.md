# フェーズC: 編集フロー

---

## ステップ9(編集対象の選択)

**条件**: モード選択で「既存エージェントを編集する」が選択された場合

**入力**: エージェント名またはパス（指定されていれば）

**AI実施タスク**:
- 全スコープの `agents/` を走査し、エージェントファイルを一覧化する:
  - 各ファイルのフロントマター（name / description / model）を読み取る
  - スコープ・カテゴリフォルダ・パスを付与する
- エージェント名やパスが既に指定されている場合は一致するものを特定し、このステップの選択をスキップしてステップ10へ進む
- 見つからない場合はエラーメッセージと一覧を表示する

**出力**:
- エージェント一覧テーブル:

  | No. | 名前 | 概要 | スコープ | パス |
  |---|---|---|---|---|
  | 1 | cost-conscious | 開発・運用コストの最小化を優先 | プラグイン(idea) | `agents/implementation/cost-conscious.md` |
  | 2 | model-selection | 用途からモデルを比較・推薦 | プラグイン(idea) | `agents/ai-dev/model-selection.md` |
  | ... | | | | |

- 「どのエージェントを編集しますか？（番号またはエージェント名で指定）」

**選択肢**:
  1. エージェントを番号または名前で指定する → ステップ10へ
  2. スコープで絞り込む（ユーザー / プロジェクト / プラグイン名）
  3. 戻る → ステップ1（モード選択）へ
  4. その他（入力してください）

**補足**:
- エージェント数が多い場合はスコープやカテゴリフォルダで絞り込みを提案する
- リクエストで「ideaのcost-consciousを修正して」のように対象が明確な場合、一覧表示をスキップしてステップ10に直接進む

## ステップ10(編集内容の確認・実行)

**条件**: 編集対象のエージェントが特定された後

**入力**: 対象エージェントのファイルパス

**AI実施タスク**:
- 対象エージェントファイルを読み込み、全内容を表示する
- フロントマターとボディをセクション分けして表示し、どこを編集するか明確にする
- ユーザーの修正指示を受けて変更案を生成する
- 変更前と変更後の差分を見やすく表示する（変更箇所をハイライト）
- ユーザーが承認したらファイルを更新する
- 配置先スキルの SKILL.md にエージェント一覧テーブルがある場合、description 等の変更に合わせて更新が必要か確認する
- `~/.claude/skill-memory/agent-wizard/` に編集内容を記録

**出力**:
- エージェントファイルの現在の全内容（フロントマター / ボディのセクション分け）
- 「どの部分を修正しますか？」

**選択肢**:
  1. フロントマターを変更する（フィールドと値を指定）
  2. ボディ（役割・コアバリュー・推論スタイル等）を変更する
  3. 関連リンクを追加・修正する
  4. 回答形式を変更する
  5. 修正完了 → ステップ15（サマリー出力）へ
  6. 別のエージェントも編集する → ステップ9へ
  7. その他（入力してください）

**補足**:
- 修正は複数回ループ可能。選択肢5「修正完了」を選ぶまでこのステップを繰り返す
- ファイル書き込みはユーザーが変更内容を承認した後に行う。差分表示→承認→書き込みの流れを守る
