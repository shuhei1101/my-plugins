# フェーズ2: MCP情報取得

---

## ステップ2（情報源選択）

**条件**: メインメニューで「最新MCP情報を取得する」が選択された場合、またはおすすめ探索時に`mcp-catalog/`が空で誘導された場合

**入力**: -

**AI実施タスク**:
- 利用可能な情報源の一覧を、おすすめ順に特徴付きで提示する
- `mcp-catalog/` の現在の状態を確認し、最終更新日時があれば表示する
- 情報源の詳細は `references/sources_guide.md` を参照する

**出力**:
- `mcp-catalog/` の状態（「データなし」or「最終更新: YYYY-MM-DD」）
- 情報源の一覧:

| No. | 情報源 | おすすめ | 特徴 |
|-----|--------|----------|------|
| 1 | 公式レジストリAPI | ⭐ | Anthropic公式。400+サーバーのメタデータを網羅 |
| 2 | FastMCP | ⭐ | 人気度・月間トレンド・インストール数がわかる |
| 3 | GitHub | | 公式リポジトリのREADME・スター数から情報取得 |
| 4 | npm検索 | | パッケージ名・ダウンロード数で検索 |
| 5 | Context7 | | 最新ドキュメント・コード例の取得に強い |

**選択肢**:
1. おすすめの情報源で取得する（1+2） → ステップ3へ
2. 情報源を選択する（番号で指定、複数可） → ステップ3へ
3. すべての情報源から取得する → ステップ3へ
4. 戻る → ステップ1へ
5. その他（入力してください）

**補足**:
- 初回は「おすすめの情報源で取得する」をデフォルトとして推奨
- 複数選択の場合はカンマ区切り（例: 「1,3,5」）

---

## ステップ3（サブエージェント並列検索）

**条件**: ステップ2で情報源が選択された場合

**入力**: 選択された情報源のリスト

**AI実施タスク**:
- 選択された情報源ごとにサブエージェントを並列起動:
  - 各情報源の取得方法・URLは `references/sources_guide.md` を参照
  - **公式レジストリAPI**: WebFetchで `https://api.anthropic.com/mcp-registry/v0/servers` からサーバー一覧を取得
  - **FastMCP**: WebFetchで `https://fastmcp.me` から人気ランキング・トレンド情報を取得
  - **GitHub**: WebFetchで `https://github.com/modelcontextprotocol/servers` のREADMEからサーバー一覧を取得
  - **npm検索**: Bashで `npm search mcp-server` 等でパッケージを検索
  - **Context7**: Context7 MCPが利用可能なら最新ドキュメントを取得
- 各サブエージェントの必読情報:
  - 取得したデータを `| MCP名 | 概要 | URL |` の形式に整形すること
  - 分野カテゴリの判定基準は `references/category_rules.md` を参照
- 全サブエージェントの完了を待ち、結果を統合する
- 統合時に重複を除去し、分野ごとに分類する

**出力**:
- 取得件数サマリー（「公式レジストリから120件、FastMCPから50件取得しました」等）
- 分野別の取得結果をプレビュー表示（各分野の上位3件程度）:

```
### 開発ツール系（25件）
| MCP名 | 概要 | URL |
|--------|------|-----|
| playwright | ブラウザ自動化 | https://... |
| github | リポジトリ・PR・Issue統合 | https://... |
| ... | ... | ... |

### データ系（18件）
...
```

**選択肢**:
1. この結果でOK → ステップ4（結果表示と次のアクション）へ
2. 追加の情報源でもう一度検索する → ステップ2へ
3. 特定の分野だけ詳しく見る（分野名を指定）
4. その他（入力してください）

**補足**:
- サブエージェントは必ず同一ターンで並列起動する。1つの完了を待たない
- WebFetchでエラーが出た情報源はスキップし、成功した情報源の結果のみ表示する
- Context7 MCPが未インストールの場合はその旨を通知し、他の情報源で続行する

---

## ステップ4（結果表示と次のアクション）

**条件**: ステップ3で検索結果の統合が完了した場合

**入力**: ステップ3の統合結果

**AI実施タスク**:
- 取得結果の全体サマリーを表示する
- `mcp-catalog/` への保存を提案する
  - 既存データがある場合: 差分（新規追加・更新・削除されたMCP）を表示
  - 新規の場合: 分野別ファイルの作成一覧を表示
- ユーザーの承認を得てから保存を実行する
- 保存完了後、結果メッセージを表示する（例: 「✓ mcp-catalog/development.md に 25件保存しました」）
- 各分野ファイルの先頭にメタ情報を以下の形式で記録する:
  ```markdown
  # 開発ツール系MCP
  <!-- 最終更新: 2026-04-02 -->
  <!-- 情報源: 公式レジストリAPI, FastMCP -->

  | MCP名 | 概要 | URL |
  |--------|------|-----|
  | ... | ... | ... |
  ```

**出力**:
- 全体サマリー:
  ```
  取得結果: 合計 XX 件（重複除去済み）
  - 開発ツール系: XX件
  - データ系: XX件
  - 検索系: XX件
  - 生産性系: XX件
  - クラウド系: XX件
  - その他: XX件
  ```
- 保存先の確認:
  ```
  以下のファイルに保存します:
  mcp-catalog/development.md (XX件)
  mcp-catalog/data.md (XX件)
  ...
  ```

**選択肢**:
1. カタログに保存して、おすすめMCPを探す → 保存実行後ステップ5へ
2. カタログに保存して、メインメニューに戻る → 保存実行後ステップ1へ
3. カタログに保存のみ → 保存実行後ステップ13へ
4. 保存せずにおすすめMCPを探す → ステップ5へ（取得結果をそのまま使用）
5. その他（入力してください）

**補足**:
- 保存時、各分野ファイルの先頭にメタ情報を記録する（`最終更新: YYYY-MM-DD`、`情報源: {使用した情報源}`）
- 既存データとのマージ時は、URL基準で重複判定する
- 分野の自動分類が不適切な場合、ユーザーが手動で修正できる選択肢も用意する
