# フェーズ3: おすすめMCP探索

---

## ステップ5（調査方法選択）

**条件**: メインメニューで「おすすめMCPを探す」が選択された場合、またはステップ4から遷移した場合

**入力**: -

**AI実施タスク**:
- `mcp-catalog/` の状態を確認する
  - データがない場合: 「まず最新情報を取得しましょう」とステップ2への誘導を提案
  - データがある場合: 最終更新日時を表示し、調査方法の選択肢を提示
- 各調査方法の説明を添える

**出力**:
- `mcp-catalog/` の状態（「最終更新: YYYY-MM-DD、合計XX件」）
- 調査方法の説明

**選択肢**:
1. SKILL.mdから探索する（スキルのファイルパスを指定してください） → ステップ6へ
2. 開発中のコードから探索する（対象ディレクトリを指定してください） → ステップ6へ
3. トレンド・人気ランキングを見る → ステップ6へ
4. ランダムで出会う（おまけ） → ステップ6へ
5. 戻る → ステップ1へ
6. その他（入力してください）

**補足**:
- 選択肢1を選んだ場合、ファイルパスの入力を待つ。未入力ならカレントディレクトリのSKILL.mdを探す
- 選択肢2を選んだ場合、対象ディレクトリの入力を待つ。未入力ならカレントディレクトリを使用
- `mcp-catalog/` が空の場合でも選択肢3・4は利用可能（取得済みデータなしで直接WebFetch等する）

---

## ステップ6（おすすめ結果表示）

**条件**: ステップ5で調査方法が選択された場合

**入力**: 調査方法 + 必要に応じてファイルパス/ディレクトリ

**AI実施タスク**:
- 調査方法に応じてサブエージェントを起動:
  - **SKILL.mdから探索**:
    - サブエージェント: 指定されたSKILL.mdを読み込み、スキルの機能・使用技術・依存ツールを分析
    - `mcp-catalog/` のデータと照合し、**まだカタログに登録されていないMCP**の中からスキルの機能を強化・補完できるものを抽出
    - 必読情報: 対象SKILL.md、`mcp-catalog/` 全ファイル
  - **開発コードから探索**:
    - サブエージェント: 対象ディレクトリのコードを走査し、使用言語・フレームワーク・外部サービス・API呼び出しを分析
    - `mcp-catalog/` のデータと照合し、**まだカタログに登録されていないMCP**の中から開発を効率化できるものを抽出
    - 必読情報: 対象ディレクトリ構成、package.json等の依存ファイル、`mcp-catalog/` 全ファイル
  - **トレンド・人気ランキング**:
    - `mcp-catalog/` 内のデータから人気度・汎用性の高い**カタログ未登録MCP**を上位表示
    - データがない場合はWebFetchで直接FastMCPから取得
  - **ランダム**:
    - `mcp-catalog/` から**カタログ未登録MCP**をランダムに5〜10件抽出して表示
    - データがない場合はWebFetchで公式レジストリから取得してランダム表示
- 結果をおすすめ度順に整理して表示
- 検索対象から`mcp-catalog/`に既存のMCPを除外する

**出力**:
- 調査方法と対象の確認（「SKILL.md `plugins/idea/skills/idea/SKILL.md` からの探索結果」等）
- 「カタログ未登録のMCPから探索しました」と明示
- おすすめMCP一覧:

| No. | MCP名 | おすすめ度 | 概要 | 理由 | URL |
|-----|--------|-----------|------|------|-----|
| 1 | xxx | ⭐⭐⭐ | ... | スキルの○○機能を強化 | https://... |
| 2 | yyy | ⭐⭐ | ... | △△の処理を効率化 | https://... |
| ... | ... | ... | ... | ... | ... |

**選択肢**:
1. MCPを選んで登録する（番号で指定） → ステップ7へ（選択したMCP情報を引き継ぐ）
2. すべてのMCPをカタログに保存する → `mcp-catalog/` に追加保存してステップ5へ
3. 別の調査方法で探す → ステップ5へ
4. 詳細を見る（番号で指定） → 選択したMCPの詳細情報を表示して再度この選択肢へ
5. メインメニューに戻る → ステップ1へ
6. その他（入力してください）

**補足**:
- 検索は常にカタログ既存データを除外し、新しいMCPのみ結果に含める
- SKILL.mdからの探索では「なぜこのMCPがこのスキルに合うか」の理由を必ず添える
- 開発コードからの探索では、検出した技術スタック（言語・フレームワーク等）を先に表示してからおすすめを提示する
- 選択肢1で登録フローに進む場合、MCP名・URL・設定コマンド例を自動で引き継ぐ
- 選択肢2で保存する場合、分野カテゴリを自動判定して適切なファイルに追記する
