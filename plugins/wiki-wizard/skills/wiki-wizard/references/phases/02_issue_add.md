# フェーズ2: 新規 Issue の追加

`wiki/Issues.md` に新規 Issue（`ISSUE-XXX`）を追加するフェーズ。連番採番・選択肢整理を支援する。

---

## ステップ2.1: Issue タイトル・背景の入力

**条件**: フェーズ2 開始時

**入力**:
- ユーザーが起動時に渡した「Issue 追加：◯◯について」などのテキスト（あれば自動でタイトル候補化）
- なければユーザーに「どんな Issue を追加したいですか？」と質問

**AI実施タスク**:
- 起動時引数からタイトル候補を抽出（あれば）
- ユーザーに以下を質問（多角度質問形式）:

```
Q1. Issue タイトルは？（短く、何を決めるかが分かる形で）
→ {引数からの推測候補があれば提示}

Q2. 背景は？（なぜこのイシューが必要か、1〜3 行）
→ 

Q3. 検討中の選択肢はありますか？（A / B / C のように 2〜3 個。なければスキップで AI が案出し）
→ 

💡 **Tips**: 入力時は `/copy` コマンドを使用して入力フォームを活用してください
```

**出力**: 上記質問

**選択肢**: 自由入力（質問への回答）

**補足**:
- ユーザーが選択肢をスキップした場合、ステップ2.4 で AI が選択肢案を生成

---

## ステップ2.2: 連番採番

**条件**: ステップ2.1 完了後

**入力**: なし（自動取得）

**AI実施タスク**:
- Grep ツールで `wiki/Issues.md` と `wiki/イシュー履歴.md` から `ISSUE-(\d+)` パターンを抽出

```
Grep tool:
  pattern: "ISSUE-(\d+)"
  path: "wiki/"
  output_mode: "content"
```

- 抽出した数値の最大値 + 1 を新規 Issue 番号として決定
- 両ファイルが存在しない / 一度も Issue がない場合は `ISSUE-001` から開始

**出力**: なし（内部処理。次のステップで採番結果を提示）

**選択肢**: なし（自動でステップ2.3 へ）

---

## ステップ2.3: 関連ドキュメントの特定

**条件**: ステップ2.2 完了後

**入力**: ステップ2.1 で得たタイトル・背景

**AI実施タスク**:
- `wiki/Home.md` を Read してドキュメント一覧を把握する
- Issue のタイトル・背景のキーワードで `wiki/` 配下を Grep し、関連する記述を含むドキュメントを特定する

```
Grep tool:
  pattern: "{タイトル・背景のキーワード}"
  path: "wiki/"
  output_mode: "files_with_matches"
```

- 特定した各ドキュメントについて「関連する理由」を 1 行で整理する
- 関連ドキュメントが見つからない場合は「なし」とする（無理に探さない）

**出力**: なし（内部処理。次のステップでドラフトに反映）

**選択肢**: なし（自動でステップ2.4 へ）

---

## ステップ2.4: 選択肢の整理（AI による補助）

**条件**: ステップ2.3 完了後

**入力**: ステップ2.1 で得たタイトル・背景・選択肢候補、ステップ2.3 で特定した関連ドキュメント

**AI実施タスク**:
- ユーザーが選択肢を提示済み → そのまま整形
- ユーザーが選択肢をスキップ → AI が背景から選択肢案を 2〜3 個生成し、推奨方式（採用案 + 理由）も提案
- `examples/issue_example.md` のフォーマットに従って整形
- ステップ2.3 で特定した関連ドキュメントを `**関連ドキュメント**` セクションとして末尾に追加する

**出力**: 整形済みの Issue ドラフトを提示

```
以下の内容で `wiki/Issues.md` に追記します。

---

## ISSUE-{採番} : {タイトル} 未決定

**背景**: {背景文}

**選択肢**:
- **A**: {選択肢A}
- **B**: {選択肢B}
- **C**: {選択肢C}

**推奨方式**: {A/B/C} — {理由}

**関連ドキュメント**:
- [{ドキュメント名}]({ファイル名}.md) — {関連する理由}
- [{ドキュメント名}]({ファイル名}.md) — {関連する理由}

---

1. この内容で追記する
2. 修正する（どこを直すか入力）
3. 選択肢を追加する
4. キャンセル
5. その他（入力してください）

[████████████░░░░░░░░] ステップ2.4 / 全6フェーズ
```

**選択肢**:
1. → ステップ2.5 へ
2. → ユーザーに修正内容を入力してもらい、再度ドラフト提示（このステップを繰り返す）
3. → 選択肢を追加してドラフト再生成
4. → フェーズ終了
5. その他

**補足**:
- ユーザーが「Q3 にサブ Q を含めたい」と入れた場合は `### ISSUE-XXX-1`, `### ISSUE-XXX-2` のサブ番号で展開する
- 関連ドキュメントが存在しない場合は `**関連ドキュメント**` セクション自体を省略してよい

---

## ステップ2.5: Issues.md への追記実行

**条件**: ステップ2.4 で確定後

**入力**: なし

**AI実施タスク**:
- `wiki/Issues.md` の末尾（または適切なカテゴリセクション）に追記する内容を確定
- サブエージェント（バックグラウンド）に書き込み委譲

**サブエージェント起動例**:

```
Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "ISSUE-{採番} を Issues.md に追記"
  prompt:
    """
    {プロジェクトルート}/wiki/Issues.md の末尾に、以下の Issue を追記してください。

    ===
    {ステップ2.4 で確定したドラフト全文}
    ===

    追記前の確認事項:
    1. Issues.md の末尾構造（最後の Issue の終了マーカー、空行など）を確認し、適切な位置に挿入
    2. 既に同じ ISSUE 番号がないか Grep で確認（あれば中断して報告）
    3. 完了後、追記したファイルパスと ISSUE 番号を 1 行で報告
    """
```

**出力**:

```
✅ ISSUE-{採番} 「{タイトル}」を Issues.md に追記するサブエージェントをバックグラウンドで起動しました。
完了通知が届いたら確認します。

引き続き Issue を追加しますか？

1. 続けて新規 Issue を追加（ステップ2.1 に戻る）
2. メニューに戻る
3. 終了
4. その他（入力してください）

[████████████████████] ステップ2.5 / 全6フェーズ
```

**選択肢**:
1. → ステップ2.1 へ戻ってループ
2. → フェーズ0 メニューへ
3. → 終了
4. その他

**補足**:
- バックグラウンド起動のため、メインは完了を待たずに次のユーザー入力を受け付ける
- 完了通知が届いたら「ISSUE-XXX を追加しました」と簡潔に報告するのみ
