# フェーズ3: Issue を決定済みにして転記・履歴追記

ユーザーが Issue を決定したら、以下を **並列バックグラウンド** で自動実行する:

A. 決定内容を **機能別ドキュメント** に転記
B. `Issues.md` から該当 Issue を削除
C. `イシュー履歴.md` の最下部に履歴追記
D. `wiki.md` のリンク集を必要に応じて更新

---

## ステップ3.1: 決定対象 Issue の特定

**条件**: フェーズ3 開始時

**入力**:
- ユーザーが起動時に「ISSUE-XXX を決定」と渡した場合 → 対象は確定
- 渡されていない場合 → `Issues.md` の Issue 一覧を表示し、選んでもらう

**AI実施タスク**:
- `wiki/Issues.md` を Read で読み込む
- `ISSUE-(\d+)` パターンを抽出し、各 Issue のタイトル・ステータスを一覧化

**出力（一覧表示）**:

```
現在 Issues.md に登録されている未決定 Issue:

1. ISSUE-007: 直列処理の失敗処理（検討中）
2. ISSUE-009: ◯◯の閾値設定（未決定）
3. ISSUE-012: △△モードの仕様（保留）
...

どの Issue を決定しますか？

1〜N. 上記から選択
N+1. その他（番号を直接入力 or キャンセル）

[████░░░░░░░░░░░░░░░░] ステップ3.1 / 全6フェーズ
```

**選択肢**: 番号選択 or 自由入力

**補足**:
- 一覧が10件以上ある場合は、ステータス別（未決定 / 検討中 / 保留）にグルーピングして提示
- 「キャンセル」を選んだら、フェーズ0 メニューに戻る

---

## ステップ3.2: 採用選択肢と決定理由の確認

**条件**: ステップ3.1 で対象 Issue を特定後

**入力**: ユーザーの決定内容

**AI実施タスク**:
- 対象 Issue の本文を抽出して提示
- ユーザーに以下を質問:

**出力**:

```
ISSUE-{番号}: {タイトル}

【背景】
{背景文}

【選択肢】
- A: ...
- B: ...
- C: ...

【推奨方式】{A/B/C} — {理由}

---

Q1. 採用する選択肢は？
→ (A / B / C / カスタム)

Q2. 採用理由は？（推奨方式のままで良いなら「推奨通り」と回答）
→ 

Q3. 決定日は？（デフォルト: 今日 = {YYYY-MM-DD}）
→ 

💡 **Tips**: 入力時は `/copy` コマンドを使用して入力フォームを活用してください

[████████░░░░░░░░░░░░] ステップ3.2 / 全6フェーズ
```

**選択肢**: 自由入力（Q1〜Q3 への回答）

**補足**:
- カスタム選択肢の場合は、Q2 で具体的内容を必須化
- 「推奨通り」の場合は AI が推奨方式の理由をそのまま使用

---

## ステップ3.3: 転記先機能別ドキュメントの判定

**条件**: ステップ3.2 完了後

**入力**: 採用選択肢と決定理由

**AI実施タスク**:
- マスター判定原則（`references/workflow_rules.md` 参照）に従い、転記先候補を提案
- 既存 `wiki/` 配下のドキュメントを Glob で取得し、Issue の内容にマッチしそうなファイル名を絞り込み
- 該当ファイルがない場合は新規ファイル作成を提案

**出力**:

```
転記先ドキュメントの候補:

1. {推奨ファイル名}.md（既存）— 関連度: 高
   → このファイルは {内容の概要}。決定内容を「{推測されたセクション名}」セクションとして追記
2. {次点候補}.md（既存）
3. 新規ファイル作成（ファイル名を入力）
4. その他（入力してください）

[████████████░░░░░░░░] ステップ3.3 / 全6フェーズ
```

**選択肢**:
1. → 推奨ファイルに転記
2. → 次点ファイルに転記
3. → 新規ファイル名を入力後、転記
4. その他

**補足**:
- マスター判定原則の例:
  - 全モード共通仕様 → `共通仕様.md`
  - 機能別仕様 → 該当機能のドキュメント
  - AI 構成・LLM 関連 → `AI構成.md`
  - API 仕様 → 各 API ドキュメント

---

## ステップ3.4: 重複チェック

**条件**: ステップ3.3 で転記先確定後

**入力**: なし

**AI実施タスク**:
- 決定内容のキーワード（タイトルから抽出）を Grep で `wiki/` 配下から全文検索
- 既に同等の記載がある場合は、参照リンク化を提案

**出力（重複あり）**:

```
⚠ 以下のドキュメントに関連する記載が見つかりました:

- {ドキュメントA}.md:{行番号}: 「{該当行の抜粋}」
- {ドキュメントB}.md:{行番号}: 「{該当行の抜粋}」

対応:
1. マスター（{転記先}.md）に詳細を集約し、他は参照リンクに置き換える（推奨）
2. 重複を許容してそのまま転記する
3. 内容が異なるので別セクションとして転記する
4. その他（入力してください）

[████████████████░░░░] ステップ3.4 / 全6フェーズ
```

**出力（重複なし）**:

```
✅ 重複なし。転記して問題ありません。
ステップ3.5 で並列バックグラウンド処理を起動します。
```

**選択肢**: 上記参照（重複ありの場合のみ）

**補足**:
- 「参照リンクに置き換える」を選んだ場合は、ステップ3.5 のサブエージェント起動内容に「他ドキュメントの該当箇所を参照リンクに書き換える」タスクを追加

---

## ステップ3.5: 並列サブエージェント起動

**条件**: ステップ3.4 完了後

**入力**: なし

**AI実施タスク**: 以下4つのタスクを **並列バックグラウンド** で起動

### タスクA: 機能別ドキュメントへの追記

```
Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "ISSUE-{番号} 決定内容を {転記先}.md に追記"
  prompt:
    """
    {プロジェクトルート}/wiki/{転記先}.md に以下の決定内容を追記してください。

    【追記内容】
    ## {セクションタイトル}

    {決定内容の本文}

    **決定経緯**:
    - 元の Issue: ISSUE-{番号} 「{タイトル}」
    - 採用選択肢: {採用案}
    - 採用理由: {理由}
    - 決定日: {YYYY-MM-DD}
    - 検討した選択肢:
      - A: {選択肢A}
      - B: {選択肢B}
      - C: {選択肢C}

    【ルール】
    1. 適切なセクション位置に挿入（既存の構造を尊重）
    2. 重複チェック: Grep で `wiki/` 配下の同等記載を確認し、あれば参照リンクに統一
    3. ファイル末尾の `**最終更新**` 行を以下の形式で更新する:
       `**最終更新**: {YYYY-MM-DD} — ISSUE-{番号} 決定に伴い{変更内容の概要}`
    4. 完了後、書き込んだファイルパスとセクション名を報告
    """
```

### タスクB: Issues.md から該当 Issue 削除

```
Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "ISSUE-{番号} を Issues.md から削除"
  prompt:
    """
    {プロジェクトルート}/wiki/Issues.md から ISSUE-{番号} のセクション全体（## ISSUE-XXX から次の ## or --- まで）を削除してください。

    削除前の確認:
    1. 該当 Issue の本文を Grep で特定
    2. セクション境界を慎重に判定（次の Issue の開始または ファイル末尾まで）
    3. 削除後、ファイル内に空行が連続しないよう整形
    4. 完了後、削除した範囲（行数）を報告
    """
```

### タスクC: イシュー履歴.md の最下部に履歴追記

```
Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "ISSUE-{番号} を イシュー履歴.md に追記"
  prompt:
    """
    {プロジェクトルート}/wiki/イシュー履歴.md の最下部に、以下の履歴を追記してください。

    ## ISSUE-{番号}: {タイトル}

    - **質問内容**: {元の質問の概要、1〜2 行}
    - **決定した内容**: {採用選択肢と理由の要約、1〜3 行}
    - **決定日**: {YYYY-MM-DD}
    - **転記先**: [{転記先ドキュメント名}](wiki/{転記先ファイル名}.md#{セクションID})

    ---

    【ルール】
    1. ファイルの最末尾に追記（既存履歴の後）
    2. セクション ID は Markdown 自動生成のものを使用（日本語混在時は URL エンコード）
    3. 完了後、追記したパスと ISSUE 番号を報告
    """
```

### タスクD: wiki.md のリンク集を必要に応じて更新

```
条件: 新規ファイル作成 or 既存ファイルへの大きなセクション追加が発生した場合のみ起動

Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "wiki.md のリンク集を更新"
  prompt:
    """
    {プロジェクトルート}/wiki/Home.md のリンク集を最新化してください。

    【今回の変更】
    - 新規ファイル: {ファイル名}（あれば）
    - 既存ファイルの新規セクション: {ファイル名}#{セクション} （あれば）

    【ルール】
    1. wiki.md の構造を保ち、適切なセクションに追加
    2. ファイル名から内容を推測し、配信モード / システム設計 / API 仕様 / 参考資料 などに分類
    3. 既存のリンクと重複しないこと
    4. 完了後、追加したリンクを報告
    """
```

**出力**:

```
✅ 4つのサブエージェントをバックグラウンドで並列起動しました:

A. {転記先}.md への追記
B. wiki/Issues.md から ISSUE-{番号} 削除
C. wiki/イシュー履歴.md への履歴追記
D. wiki/Home.md のリンク集更新（{該当 / 不要}）

完了通知が順次届きます。引き続き次のアクションへ進めます。

1. 続けて別の Issue を決定（ステップ3.1 に戻る）
2. 新規 Issue を追加（フェーズ2 へ）
3. メニューに戻る
4. 終了
5. その他（入力してください）

[████████████████████] ステップ3.5 / 全6フェーズ
```

**選択肢**:
1. → ステップ3.1 へ戻ってループ
2. → フェーズ2 へ
3. → フェーズ0 メニューへ
4. → 終了
5. その他

**補足**:
- 4つのタスクは互いに独立しているため並列起動が安全
- ただし、タスクB（Issues.md 削除）とタスクA（転記）は論理的に「転記が成功してから削除」が望ましい場合もある。**重要な決定や転記内容が大きい場合**は、タスクA → タスクB を直列にする選択肢を提供しても良い
- 並列起動後、メインは完了を待たず次の入力を受け付ける（バックグラウンド完了通知が届いたら簡潔に確認のみ）
