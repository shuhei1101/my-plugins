# フェーズ0: メインメニュー

スキル起動時のエントリーポイント。引数の有無で以下に分岐:

- 引数なし → メニューを表示してユーザーに選んでもらう
- 引数あり → ユーザーの意図を解釈し、該当フェーズへ直接ジャンプ

---

## ステップ0.1: 起動コンテキスト判定

**条件**: `/wiki-wizard` または `/wiki-wizard:wiki-wizard` で起動

**入力**: ユーザーが起動時に渡した引数（自由テキスト）

**AI実施タスク**:
- 引数の有無を判定
- 引数がある場合は、以下のキーワード辞書で意図を解釈し、該当フェーズへジャンプ判定

| 引数キーワード例 | ジャンプ先フェーズ |
| --- | --- |
| 「初期化」「Wiki 作って」「Wiki 構築」「セットアップ」 | フェーズ1 |
| 「Issue 追加」「新規 Issue」「ISSUE 立てて」 | フェーズ2 |
| 「Issue 決定」「ISSUE-XXX 決まった」「決定して履歴に」 | フェーズ3 |
| 「重複チェック」「ドキュメント整理」「整合性チェック」 | フェーズ4 |
| 「home.md 更新」「リンク集 最新化」「ナビ更新」 | フェーズ5 |
| 「CLAUDE.md にルール追加」 | フェーズ1.5（独立呼び出し） |

- 判定がつかない引数の場合は、メニューを表示し、ユーザーが意図したフェーズを選んでもらう

**出力**:
- 引数あり・判定可 → 該当フェーズへ直接ジャンプ（その旨を一言報告: 「Issue 追加フローに入ります」など）
- 引数なし or 判定不可 → ステップ0.2 へ

---

## ステップ0.2: メニュー選択

**条件**: ステップ0.1 で引数なし or 判定不可と判定された場合

**入力**: なし（メニュー表示が目的）

**AI実施タスク**:
- 現在のプロジェクトの `wiki/` フォルダの有無を Glob で確認
  - ない場合: 選択肢1（初期化）を強く推奨
  - ある場合: 通常のメニューを表示

**出力**:

```
🧙 wiki-wizard — ドキュメント Wiki 構築・運用ウィザード

何をしますか？

1. Wiki 構造を初期化（新規プロジェクト用 / 既存プロジェクトへの後付けも対応）
2. 新規 Issue を追加
3. Issue を決定済みにして転記・履歴追記
4. ドキュメント間の重複チェック
5. home.md のリンク集を最新化
6. CLAUDE.md にドキュメント管理ルールを追記
7. その他（入力してください）

[████░░░░░░░░░░░░░░░░] ステップ0.2 / 全6フェーズ
```

**選択肢**:
1. → フェーズ1 へ
2. → フェーズ2 へ
3. → フェーズ3 へ
4. → フェーズ4 へ
5. → フェーズ5 へ
6. → フェーズ1.5（独立呼び出し）へ
7. その他（入力してください）

**補足**:
- 既存プロジェクト（`wiki/home.md` がすでに存在）の場合は、選択肢の文言を「Wiki 構造を初期化（既存 Wiki の補完・更新）」に変更する
- どの選択肢にも当てはまらない自由入力の場合は、ステップ0.1 のキーワード辞書で再判定する
