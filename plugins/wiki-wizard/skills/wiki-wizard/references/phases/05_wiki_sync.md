# フェーズ5: wiki.md のリンク集を最新化

`docs/` 配下のファイル一覧を Glob で取得し、`docs/wiki.md` のリンク集と差分を取って、追加・削除・リネーム・並び替えを反映する。

---

## ステップ5.1: 既存 docs/ のスキャン

**条件**: フェーズ5 開始時

**入力**: なし

**AI実施タスク**:
- `docs/*.md` を Glob で全取得
- 各ファイルの先頭数行を Read で取得し、タイトル（H1）と概要を抽出
- `docs/wiki.md` を Read で取得

**出力**: スキャン結果のサマリ

```
📁 docs/ スキャン結果:

存在するドキュメント数: N 個
- {ファイル名1}.md — {推測されたタイトル}
- {ファイル名2}.md — {推測されたタイトル}
- ...

wiki.md に登録済みリンク数: M 個

→ 差分検出をステップ5.2 で実行します。

[████░░░░░░░░░░░░░░░░] ステップ5.1 / 全6フェーズ
```

**選択肢**: 自動でステップ5.2 へ

**補足**:
- `wiki.md` `Issues.md` `イシュー履歴.md` はリンク集の対象から除外（または「主要ドキュメント」セクションに分離）

---

## ステップ5.2: 差分検出

**条件**: ステップ5.1 完了後

**入力**: なし

**AI実施タスク**:
- 「docs/ 配下のファイル一覧」と「wiki.md 内のリンク先一覧」を突合
- 以下を分類:
  - **追加（A）**: ファイルがあるが wiki.md にリンクなし
  - **削除（D）**: wiki.md にリンクがあるがファイル不在
  - **リネーム候補（R）**: wiki.md のリンク名と類似するが完全一致しないファイル

**出力**:

```
🔍 差分検出結果:

【追加候補（A）】wiki.md に登録すべき新規ドキュメント:
- 物語フック.md
- 配信フロー設計.md（追加分）

【削除候補（D）】wiki.md にリンクがあるがファイル不在:
- 古い設計書.md

【リネーム候補（R）】類似名のファイル:
- wiki.md: [VTube Studio.md] ↔ ファイル: [VTube Studio API.md]

[████████░░░░░░░░░░░░] ステップ5.2 / 全6フェーズ
```

**選択肢**: 自動でステップ5.3 へ

---

## ステップ5.3: セクション分類の提案

**条件**: ステップ5.2 完了後

**入力**: なし

**AI実施タスク**:
- 追加候補のファイル名・タイトル・概要から、所属すべきセクションを推測
- 既存 wiki.md のセクション構造（例: 配信モード / システム設計 / API 仕様 / 参考資料）を取得し、新規ファイルがどこに属するかを分類

**出力**:

```
🗂 セクション分類提案:

【配信モード仕様】（既存）
- （変更なし）

【システム設計】（既存）
- 追加: 物語フック.md
- 追加: 配信フロー設計.md

【API 仕様】（既存）
- リネーム: VTube Studio.md → VTube Studio API.md

【削除】
- 古い設計書.md（リンクのみ削除、ファイル削除はしない）

---

この内容で wiki.md を更新しますか？

1. 提案通り更新（バックグラウンド）
2. セクション分類を編集する
3. 削除候補を保留する
4. その他（入力してください）

[████████████░░░░░░░░] ステップ5.3 / 全6フェーズ
```

**選択肢**:
1. → ステップ5.4 へ
2. → ユーザーに編集内容を入力してもらい、再提案
3. → 削除を保留してステップ5.4 へ
4. その他

**補足**:
- ファイル名から分類が困難な場合、ファイルの先頭5〜10行を Read してタイトル・冒頭文から推測
- 既存セクションがない場合は新規セクション作成を提案

---

## ステップ5.4: wiki.md 再生成

**条件**: ステップ5.3 で更新内容確定後

**入力**: なし

**AI実施タスク**:
- サブエージェント（バックグラウンド）に書き込み委譲

**サブエージェント起動例**:

```
Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "wiki.md のリンク集を最新化"
  prompt:
    """
    {プロジェクトルート}/docs/wiki.md のリンク集を以下の通り更新してください。

    【追加】
    - 配信モード仕様セクションに：（変更なし）
    - システム設計セクションに：物語フック.md, 配信フロー設計.md

    【削除】
    - 古い設計書.md のリンク（ファイル自体は削除しない）

    【リネーム】
    - VTube Studio.md → VTube Studio API.md（リンクテキストとパス両方）

    【ルール】
    1. wiki.md の既存構造（セクション分類・並び順）を維持
    2. 各リンクは [{タイトル}](ファイル名.md) — {概要1行} の形式
    3. 並び順はセクション内で五十音順（または既存の規則に従う）
    4. 完了後、追加・削除・リネームしたエントリ数を報告
    """
```

**出力**:

```
✅ wiki.md 更新サブエージェントをバックグラウンドで起動しました。

完了通知が届いたら確認します。

1. メニューに戻る
2. 終了
3. その他（入力してください）

[████████████████████] ステップ5.4 / 全6フェーズ
```

**選択肢**:
1. → フェーズ0 メニューへ
2. → 終了
3. その他

**補足**:
- このフェーズはフェーズ3（Issue 決定）からも自動で呼ばれることがある（タスクD）
- 自動呼び出しの場合は、メニュー表示は省略し、ユーザーに何が更新されたかを完了通知のみで知らせる
