# フェーズ4: ドキュメント間の重複チェック

`docs/` 配下で同じ情報が複数ドキュメントに重複していないかをチェックし、マスター + 参照リンク方式に整理する。

---

## ステップ4.1: チェック対象の選択

**条件**: フェーズ4 開始時

**入力**: ユーザーの選択

**AI実施タスク**: なし（出力前の事前処理は不要）

**出力**:

```
重複チェックの対象を選んでください。

1. 特定キーワードで検索（例: 「連続失敗閾値」）
2. 特定ドキュメント全文を他ドキュメントと突合（例: YouTube配信モード.md）
3. 全ドキュメント横断スキャン（時間がかかる）
4. その他（入力してください）

[████░░░░░░░░░░░░░░░░] ステップ4.1 / 全6フェーズ
```

**選択肢**:
1. → ステップ4.2（キーワードモード）
2. → ステップ4.2（特定ドキュメントモード）
3. → ステップ4.2（全横断モード）
4. その他

---

## ステップ4.2: Grep による重複箇所の抽出

**条件**: ステップ4.1 で対象モード選択後

**入力**: モードに応じてキーワード or ファイル名を追加で取得

**AI実施タスク**:
- **キーワードモード**: ユーザーから検索キーワードを取得し、`docs/` 配下を Grep
- **特定ドキュメントモード**: 対象ドキュメントを Read し、目立つ見出し・固有名詞を抽出 → 各キーワードで Grep し他ドキュメントでの出現を集計
- **全横断モード**: `docs/` 配下の全 .md を Glob で取得し、各ファイルの主要見出し（## や ### の見出し）を抽出 → 同一見出し名や類似フレーズを Grep で交差検索

**出力（例：キーワードモード）**:

```
🔍 検索キーワード: 「連続失敗閾値」

ヒット箇所:

1. docs/共通仕様.md:42-48
   - 「連続失敗閾値」セクション（詳細あり：STT 3回, TTS 2回, LLM 2回）

2. docs/YouTube配信モード.md:120-122
   - 「エラー時の動作」内に同じ閾値の数値が直書きされている

→ 重複の可能性あり（マスター候補: 共通仕様.md）

[████████░░░░░░░░░░░░] ステップ4.2 / 全6フェーズ
```

**選択肢**: 自動でステップ4.3 へ

**補足**:
- ヒット件数が膨大な場合（>20件）は、上位N件のみ提示し、「全件を表示」選択肢を出す

---

## ステップ4.3: マスター判定 → 参照リンクへの書き換え提案

**条件**: ステップ4.2 で重複箇所を抽出後

**入力**: なし（AI が判定）

**AI実施タスク**:
- マスター判定原則（`references/workflow_rules.md` 参照）に従い、各キーワードのマスタードキュメントを判定
- 非マスター側を「参照リンク化」する書き換え案を生成

**出力（例）**:

```
🎯 マスター判定: docs/共通仕様.md（全モード共通仕様のため）

書き換え提案:

【docs/YouTube配信モード.md:120-122】

**現状**:
```
## エラー時の動作
連続失敗閾値:
- STT: 3 回
- TTS: 2 回
- LLM: 2 回
```

**書き換え後**:
```
## エラー時の動作
連続失敗閾値の詳細は [共通仕様.md - 連続失敗閾値](共通仕様.md#連続失敗閾値) を参照
```

---

1. この書き換えを適用する（バックグラウンド）
2. マスター判定を変更する（YouTube配信モード.md をマスターに）
3. 重複を許容してスキップ
4. 別の対応を入力する
5. その他（入力してください）

[████████████░░░░░░░░] ステップ4.3 / 全6フェーズ
```

**選択肢**:
1. → ステップ4.4 へ
2. → マスター判定を変更し、書き換え提案を再生成
3. → スキップ（次の重複箇所へ、または終了）
4. → ユーザー入力を反映
5. その他

**補足**:
- 重複箇所が複数ある場合は、1 つずつユーザーに確認 or 一括承認モードを提供

---

## ステップ4.4: 書き換え実行

**条件**: ステップ4.3 で書き換えを承認後

**入力**: なし

**AI実施タスク**:
- サブエージェント（バックグラウンド）に書き換え委譲

**サブエージェント起動例**:

```
Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "重複箇所を参照リンクに書き換え"
  prompt:
    """
    {プロジェクトルート}/docs/YouTube配信モード.md の {対象範囲} を以下に書き換えてください。

    【置換対象】
    {現状の文字列}

    【置換後】
    {書き換え後の文字列}

    【ルール】
    1. Edit ツールで正確に置換
    2. 周辺のフォーマット（インデント・空行）を崩さない
    3. 書き換え後、参照リンク先（共通仕様.md#連続失敗閾値）が実在するか Grep で確認
    4. 完了後、書き換えた範囲と参照リンク先の確認結果を報告
    """
```

**出力**:

```
✅ 書き換えサブエージェントをバックグラウンドで起動しました。

引き続き次のアクションへ:

1. 次の重複箇所を確認（ステップ4.2 の結果に戻る）
2. 別のキーワードで再チェック（ステップ4.1 へ）
3. メニューに戻る
4. 終了
5. その他（入力してください）

[████████████████████] ステップ4.4 / 全6フェーズ
```

**選択肢**:
1. → ステップ4.2 の結果から次の重複へ
2. → ステップ4.1 へ
3. → フェーズ0 メニューへ
4. → 終了
5. その他

**補足**:
- バックグラウンド完了通知は順次届く。メインは待たずに次の入力を受け付ける
- 一括承認モードを選んでいる場合は、すべての書き換えタスクを並列起動して一気に完了させる
