# フェーズ1: Wiki 構造の初期化

プロジェクトに Wiki 3点セット（`docs/wiki.md` / `docs/Issues.md` / `docs/イシュー履歴.md`）を構築するフェーズ。
新規プロジェクトでも、既に `docs/` がある既存プロジェクトでも対応する。

---

## ステップ1.1: 対象プロジェクトの確認

**条件**: フェーズ1 開始時

**入力**: なし（自動スキャン）

**AI実施タスク**:
- 現在の作業ディレクトリ（`Bash: pwd` または環境情報）を確認
- 以下を Glob でスキャン:
  - `docs/` フォルダの有無
  - `docs/wiki.md` の有無
  - `docs/Issues.md` の有無
  - `docs/イシュー履歴.md` の有無
  - プロジェクトルートの `README.md` の有無
  - プロジェクトルートの `CLAUDE.md` の有無

**出力**: スキャン結果を表形式で提示

```
📁 現在のプロジェクト: {プロジェクトルートパス}

| ファイル                     | 状態     |
| --------------------------- | -------- |
| docs/                       | あり/なし |
| docs/wiki.md                | あり/なし |
| docs/Issues.md              | あり/なし |
| docs/イシュー履歴.md        | あり/なし |
| README.md（ルート）         | あり/なし |
| CLAUDE.md（ルート）         | あり/なし |
| 既存ドキュメント数          | N 個      |
```

→ ステップ1.2 へ進む

**補足**:
- 既存ドキュメント数は `docs/*.md` の数をカウント（`wiki.md` `Issues.md` `イシュー履歴.md` を除く）
- 既存があっても、不足している3点セットだけを生成する方針

---

## ステップ1.2: 初期化方式の選択

**条件**: ステップ1.1 完了後

**入力**: ユーザーの選択

**AI実施タスク**: なし（出力前の事前処理は不要、ステップ1.1 のスキャン結果に基づき選択肢を出すだけ）

**出力**:

```
初期化の方式を選択してください。

1. 標準セットアップ（不足している3点セットだけを生成）
   → docs/wiki.md, docs/Issues.md, docs/イシュー履歴.md のうち、まだないファイルを作成

2. 既存ドキュメントを取り込んで wiki.md を生成
   → 既存の docs/*.md を Glob でスキャンし、ファイル名から推測したセクション分類で wiki.md を構築

3. 完全初期化（既存3点セットがあっても上書き、要注意）
   → 既存内容は失われる。バックアップを取った上で実行

4. その他（入力してください）

[██████░░░░░░░░░░░░░░] ステップ1.2 / 全6フェーズ
```

**選択肢**:
1. → ステップ1.3（標準）
2. → ステップ1.3（既存取り込みモード）
3. → ステップ1.3（完全初期化、確認ダイアログ追加）
4. その他

**補足**:
- 選択肢3を選んだ場合、ステップ1.3 の前に「本当に上書きしてよいですか？ y/n」の追加確認を必ず挟む

---

## ステップ1.3: 3点セットの生成

**条件**: ステップ1.2 で初期化方式を選択後

**入力**: なし（テンプレートから生成）

**AI実施タスク**:
- 以下のテンプレートをロードして、対象プロジェクトに合わせて軽くカスタマイズ
  - `examples/wiki_md_example.md` → `docs/wiki.md` に書き出し
  - `examples/issues_md_example.md` → `docs/Issues.md` に書き出し
  - `examples/issue_history_example.md` → `docs/イシュー履歴.md` に書き出し
- 「既存ドキュメントを取り込んで wiki.md を生成」モードの場合は、`docs/*.md` を Glob で取得し、ファイル名からセクション分類を推測して `wiki.md` のリンク集に展開
- ファイル書き込みは **サブエージェント（バックグラウンド）に委譲**

**サブエージェント起動例**:

```
Agent ツール:
  subagent_type=general-purpose
  run_in_background=true
  description: "Wiki 3点セットを生成"
  prompt:
    """
    以下の3ファイルを {プロジェクトルート}/docs/ 配下に新規作成してください。
    各ファイルの内容は examples/{xxx}_example.md をロードして、現在日付を埋めて書き出してください。

    1. docs/wiki.md ← examples/wiki_md_example.md（既存ドキュメント取り込みモードならファイル一覧を反映）
    2. docs/Issues.md ← examples/issues_md_example.md
    3. docs/イシュー履歴.md ← examples/issue_history_example.md

    現在の日付: {YYYY-MM-DD}（フロントマターや初期化日に埋める）

    完了後、何を作成したかを 1〜3 行で報告してください。
    """
```

**出力**: ユーザーには以下を報告:

```
✅ 3点セット生成のサブエージェントをバックグラウンドで起動しました。
完了通知が届いたら次のステップに進みます。
```

**選択肢**: なし（自動でステップ1.4 へ）

**補足**:
- バックグラウンド完了を待たずにステップ1.4 へ進んで良い（並列処理可能）
- ただしステップ1.4 が `wiki.md` の存在を前提とする場合は、メイン側で軽く待ち合わせる（または `wiki.md` の生成完了を確認してから進む）

---

## ステップ1.4: README.md への Wiki 入口リンク追加

**条件**: ステップ1.3 完了後（`wiki.md` が生成済み）

**入力**: なし

**AI実施タスク**:
- プロジェクトルート `README.md` の有無を確認
- ある場合: 末尾またはプロジェクト概要セクション直後に Wiki 入口リンクを追記する案を提示
- ない場合: 最小限の `README.md` を新規作成する案を提示

**出力（既存ありの場合）**:

```
プロジェクトルート README.md に以下を追記しますか？（既存内容は保持）

---
## 📚 ドキュメント

このプロジェクトのドキュメントは [docs/wiki.md](docs/wiki.md) を参照してください。

主要ドキュメント:
- [docs/Issues.md](docs/Issues.md) — 未決定事項・残課題の Issue 集
- [docs/イシュー履歴.md](docs/イシュー履歴.md) — 決定済みイシューの履歴
- [CLAUDE.md](CLAUDE.md) — 開発ガイド（あれば）
---

1. 追記する（バックグラウンド）
2. 追記内容を編集する
3. スキップする
4. その他（入力してください）

[████████████░░░░░░░░] ステップ1.4 / 全6フェーズ
```

**選択肢**:
1. → サブエージェントで追記（バックグラウンド）→ ステップ1.5 へ
2. → ユーザーに編集内容を入力してもらう → 確認後にサブエージェント起動 → ステップ1.5 へ
3. → 何もせずステップ1.5 へ
4. その他

**補足**:
- 既存 README.md に「ドキュメント」セクションが既にある場合は、上書き / マージ / スキップを選択させる
- README.md への直接リンク集追加は推奨しない（リンク集は wiki.md に集約する原則）

---

## ステップ1.5: CLAUDE.md への運用ルール追記（任意）

**条件**: ステップ1.4 完了後、または「CLAUDE.md にルール追加」で独立起動された場合

**入力**: なし

**AI実施タスク**:
- プロジェクトルート `CLAUDE.md` の有無を確認
- `references/claudemd_template.md` をロード
- 既存 CLAUDE.md がある場合: 同名セクション（「📁 ドキュメンツフォルダ構造」「📋 ドキュメント管理ルール」など）を Grep で検出
  - 重複あり → 上書き / マージ / スキップを選んでもらう
  - 重複なし → 末尾追記を提案

**出力**:

```
CLAUDE.md に wiki-wizard 運用ルール（references/claudemd_template.md の内容）を追記しますか？

現在の CLAUDE.md の状態: {あり / なし}
既存セクションとの重複: {検出結果}

1. 追記する（バックグラウンド、新規 or 末尾追記）
2. 追記内容を確認してから決める
3. スキップする
4. その他（入力してください）

[████████████████░░░░] ステップ1.5 / 全6フェーズ
```

**選択肢**:
1. → サブエージェントで追記
2. → claudemd_template.md の内容を表示し、ユーザー確認後に追記
3. → 何もせず終了
4. その他

**補足**:
- CLAUDE.md がプロジェクト固有のセクションを多く含む場合（例: Python 実装ルール、ビルド手順など）は、wiki-wizard の運用ルールセクションだけを独立した位置（例: 上部のドキュメント管理セクション）に追記する
- フェーズ1 の最終ステップ。完了後は「Wiki 初期化が完了しました」と報告して終了

---

## フェーズ1 完了時の報告

```
✅ Wiki 初期化が完了しました。

生成・更新したファイル:
- docs/wiki.md（ナビゲーションハブ）
- docs/Issues.md（未決定事項）
- docs/イシュー履歴.md（決定履歴）
- README.md（Wiki 入口リンク追加 / スキップ）
- CLAUDE.md（運用ルール追記 / スキップ）

次にやること:
- 「Issue 追加：◯◯について」と打って、最初の検討事項を登録してみてください
- 既存ドキュメントがある場合は「重複チェック」を実行して整合性を確認できます
```
