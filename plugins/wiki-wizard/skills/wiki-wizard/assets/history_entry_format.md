# 履歴追記フォーマット

`docs/イシュー履歴.md` に追記するエントリのフォーマット定義。

---

## 追記フォーマット

```markdown
## ISSUE-{番号}: {タイトル}

- **質問内容**: {元の質問の概要、1〜2 行}
- **決定した内容**: {採用した選択肢と理由の要約、1〜3 行}
- **決定日**: YYYY-MM-DD
- **転記先**: [{ドキュメント名}](ファイル名.md#{セクションID})

---
```

---

## 各フィールドの書き方

| フィールド | 内容 | 例 |
| --- | --- | --- |
| **質問内容** | Issues.md にあった質問の概要。1〜2 行で端的に | 「STT 失敗時の配信停止方針。3 回連続失敗時の動作が未定義だった」 |
| **決定した内容** | 採用した選択肢（A/B/C）と採用理由。1〜3 行 | 「B（3 回連続失敗で Standby 遷移）。一時的な失敗に耐性を持ちつつ安全状態に移行できるため」 |
| **決定日** | ISO 8601 形式（YYYY-MM-DD） | `2026-04-26` |
| **転記先** | 決定内容を書いたファイルへのリンク（セクション ID 付き） | `[共通仕様.md - 連続失敗閾値](共通仕様.md#連続失敗閾値)` |

---

## 記載例

```markdown
## ISSUE-007: 直列処理の失敗処理

- **質問内容**: 進行管理 → 応答生成 → メディア判定の直列処理で、各段階の失敗時の対応が未定義だった。
- **決定した内容**: B（段階別対応）。T1/T2 失敗はリトライ後 Standby、T3 失敗はデフォルト値で続行。致命的な失敗と軽微な失敗を区別して、不要な配信停止を避けるため。
- **決定日**: 2026-04-26
- **転記先**: [共通仕様.md - エラー処理方針](共通仕様.md#エラー処理方針)

---
```

---

## 追記位置のルール

- **必ず最下部に追記**（時系列順。最新が一番下）
- 末尾の `---` 区切りの後に続ける
- ファイルの末尾に改行を1行残す

---

## セクション ID の生成ルール

Markdown の見出しから自動生成される ID は以下のルールに従う:

- 半角英数字・ハイフンのみ残す
- 大文字 → 小文字
- スペース → ハイフン
- 日本語文字は多くの Markdown パーサーでそのまま残る（GitLab/GitHub 対応）

例:
- `## ISSUE-007: 直列処理の失敗処理` → `#issue-007-直列処理の失敗処理`
- `## 連続失敗閾値` → `#連続失敗閾値`

URL エンコードが必要な場合は手動で対応する。
