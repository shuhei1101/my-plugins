# Issues — 未決定事項・残課題の Issue 集

**役割**: GitHub Issue 的な扱い。未決定事項・検討中の事項・残課題を一元的に書き溜める。

**運用方針**:
1. 新しい検討事項発生 → このファイルに `ISSUE-XXX`（連番）として追加
2. ユーザーと打ち合わせ・判断 → 選択肢を決定
3. 決定したら → 対応する機能別ドキュメントに転記
4. 転記完了したら → このファイルから削除 + [イシュー履歴.md](イシュー履歴.md) に追記

**Issue 番号の付け方**:
- 連番（`ISSUE-001`, `ISSUE-002`, ...）
- 一度発行した番号は使い回さない（決定済みも履歴で参照可能なため）
- 新規追加時は Issues.md とイシュー履歴.md の両方から最大番号を確認 +1
