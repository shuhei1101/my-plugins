# Wiki Wizard — Wiki 構築・Issue 駆動運用ウィザード

## このスキルについて

プロジェクトのドキュメントを **Wiki 化** し、**Issue 駆動の意思決定** を運用するためのスキル。

### このスキルが解決する課題

- ドキュメントがどこにあるか分からない → `wiki/Home.md` に集約
- 未決定事項と決定事項が混在 → `Issues.md`（未決定）と `イシュー履歴.md`（決定済み）に分離
- 決定経緯が消える → 履歴ログに「質問 / 採用案 / 決定日 / 転記先」を追記型で残す
- 同じ仕様が複数ドキュメントに重複 → マスター + 参照リンク方式で一元化

### 提供する機能（メニュー）

1. **Wiki 構造を初期化**（新規プロジェクト用 / 既存プロジェクトへの後付けにも対応）
2. **新規 Issue を追加**（連番付与・選択肢提示フォーマット）
3. **Issue を決定済みにして転記・履歴追記**
4. **ドキュメント間の重複チェック**
5. **Home.md のリンク集を最新化**
6. **CLAUDE.md にドキュメント管理ルールを追記**

## 作業フロー

### フェーズ0: メインメニュー（引数なし起動時）

📁 **必読**: `references/phases/00_menu.md`

- ステップ0.1: 起動コンテキスト判定（引数あり → 該当フェーズへ直接ジャンプ / 引数なし → メニュー表示）
- ステップ0.2: メニュー選択

---

### フェーズ1: Wiki 構造の初期化

📁 **必読**: `references/phases/01_init.md`

- ステップ1.1: 対象プロジェクトの確認（既存 wiki/ の有無を Glob でスキャン）
- ステップ1.2: 初期化方式の選択（新規構築 / 既存ドキュメントを取り込み）
- ステップ1.3: `wiki/Home.md` `wiki/Issues.md` `wiki/イシュー履歴.md` の生成（記載例は `examples/` 配下）
- ステップ1.4: プロジェクトルート `README.md` への Wiki 入口リンク追加（既存内容は保持）
- ステップ1.5: `CLAUDE.md` への運用ルール追記の確認（任意）

---

### フェーズ2: 新規 Issue の追加

📁 **必読**: `references/phases/02_issue_add.md`

- ステップ2.1: Issue タイトル・背景の入力
- ステップ2.2: 連番採番（`Issues.md` と `イシュー履歴.md` の両方から最大番号を Grep で取得し +1）
- ステップ2.3: 関連ドキュメントの特定（`wiki/Home.md` 参照 + Grep で `wiki/` 配下を検索）
- ステップ2.4: 選択肢の整理（A / B / C ＋ 推奨方式 ＋ 関連ドキュメント）
- ステップ2.5: `Issues.md` への追記確認 → サブエージェント（バックグラウンド）で書き込み

---

### フェーズ3: Issue を決定済みにして転記・履歴追記

📁 **必読**: `references/phases/03_issue_decide.md`

- ステップ3.1: 決定対象 Issue の特定（番号指定 or 一覧から選択）
- ステップ3.2: 採用選択肢と決定理由の確認
- ステップ3.3: 転記先機能別ドキュメントの判定（マスター判定原則に基づく）
- ステップ3.4: 重複チェック（転記前に Grep で `wiki/` 配下を全文検索）
- ステップ3.5: 並列サブエージェント起動（バックグラウンド）
  - A: 機能別ドキュメントへの追記
  - B: `Issues.md` から該当 ISSUE 削除
  - C: `イシュー履歴.md` の最下部に追記
  - D: `Home.md` のリンク集を必要に応じて更新

---

### フェーズ4: ドキュメント間の重複チェック

📁 **必読**: `references/phases/04_dup_check.md`

- ステップ4.1: チェック対象の選択（特定キーワード / 特定ドキュメント全文 / 全ドキュメント横断）
- ステップ4.2: Grep による重複箇所の抽出（`wiki/` 配下を全文検索）
- ステップ4.3: マスター判定 → 非マスター側を参照リンクへ書き換える提案
- ステップ4.4: 確認後、サブエージェント（バックグラウンド）で書き換え実行

---

### フェーズ5: Home.md のリンク集を最新化

📁 **必読**: `references/phases/05_wiki_sync.md`

- ステップ5.1: 既存 `wiki/` を Glob でスキャンし、ファイル一覧取得
- ステップ5.2: `Home.md` 内のリンクと差分検出（追加・削除・リネーム検出）
- ステップ5.3: セクション分類の提案（配信モード / システム設計 / API 仕様 / 参考資料 など、プロジェクトに応じて）
- ステップ5.4: 更新内容を確認 → サブエージェントで `Home.md` を再生成

## 起動時の引数解釈

ユーザーが引数付きで起動した場合は、該当フェーズへ直接ジャンプする。判定例:

| ユーザー入力例                                  | ジャンプ先 |
| ----------------------------------------------- | ---------- |
| 「Wiki 作って」「Wiki 初期化」「初期化したい」 | フェーズ1  |
| 「Issue 追加：◯◯について」「新しい Issue」    | フェーズ2  |
| 「ISSUE-XXX を決定」「Issue 決まった」          | フェーズ3  |
| 「重複チェック」「ドキュメント整理」            | フェーズ4  |
| 「Home.md 更新」「リンク集を最新化」            | フェーズ5  |
| 「CLAUDE.md にルール追加」                     | フェーズ1.5（独立実行可） |

判定がつかない場合は、メニューを提示してユーザーに選んでもらう。

## サブエージェント実行原則

ファイル書き込み（ドキュメント更新・新規作成）は **基本的にサブエージェントのバックグラウンド実行に委譲** する。

- メイン（Claude Code）は判断・対話に集中
- 完了通知を待たずに次のステップに進む
- 詳細は `references/workflow_rules.md` を参照

---

## 既存プロジェクトへの適用

既に `wiki/` がある場合（例: AITuber プロジェクト）:

1. 既存の `wiki/Issues.md` `wiki/イシュー履歴.md` があれば踏襲（フォーマットを差分検出し提案のみ、上書きはしない）
2. `wiki/Home.md` を新規生成 or 更新（既存があれば差分提案）
3. プロジェクトルート `README.md` の既存内容は保持し、Wiki 入口リンクのみ追加（追記前に確認）
