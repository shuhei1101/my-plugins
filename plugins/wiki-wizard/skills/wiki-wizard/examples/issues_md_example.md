# Issues — 未決定事項・残課題の Issue 集

**役割**: GitHub Issue 的な扱い。未決定事項・検討中の事項・残課題を一元的に書き溜める。

**運用方針**:
1. 新しい検討事項発生 → このファイルに `ISSUE-XXX`（連番）として追加
2. ユーザーと打ち合わせ・判断 → 選択肢を決定
3. 決定したら → 対応する機能別ドキュメントに転記
4. 転記完了したら → このファイルから削除 + [イシュー履歴.md](イシュー履歴.md) に追記

**Issue 番号の付け方**:
- 連番（`ISSUE-001`, `ISSUE-002`, ...）
- 一度発行した番号は使い回さない
- 新規追加時は Issues.md とイシュー履歴.md の両方から最大番号を確認 +1

## ISSUE-001: STT 連続失敗時のフォールバック動作 未決定

**背景**: 音声認識（STT）が連続して失敗した場合、配信を止めるか継続するかが未定義。
軽微な失敗でも配信が完全停止すると視聴者体験が悪化するため、フォールバック方針が必要。

**選択肢**:
- **A**: 即座に Standby シーンへ遷移（シンプルだが軽微な失敗でも止まる）
- **B**: 3 回連続失敗で Standby 遷移（一時的な失敗に耐性あり）
- **C**: 失敗回数に応じた段階対応（柔軟だが実装が複雑）

**推奨方式**: B — 一時的なノイズに耐性を持ちつつ、連続失敗で確実に安全状態に移行できる。起動時設定で閾値を変更可能にすれば柔軟性も確保できる。

**関連ドキュメント**:
- [共通仕様.md](共通仕様.md) — 連続失敗閾値・エラー時の動作が記載されている
- [配信フロー設計.md](配信フロー設計.md) — Standby シーン遷移フローの定義がある
