# ワークフロー基本原則

このスキルは対話形式（ステップ駆動型）スキル。以下の原則に従ってユーザーと対話する。

## 基本原則

- 各ステップの最後に**必ず選択肢を提示**する
- ユーザーは選択肢を選んで会話を進める
- 選択肢以外の自由入力があった場合、AIが最も近い選択肢を判定する
- どの選択肢にも当てはまらない場合、再選択を促すか「フローから外れますがよろしいですか？」と確認する

## 選択肢の順序ルール

- 1番目: 正規ルート（OKで次に進む、この内容で確定、など）
- 2番目以降: 修正、やり直し、戻るなど
- 最後: その他（入力してください）

## 進捗バー

各ステップの選択肢の下に1行空けて進捗バーを表示する:

```
[██████░░░░░░░░░░░░░░] フェーズX.Y / 全4フェーズ
```

- バー全体幅は20文字（█ と ░ の合計）
- フェーズは1〜4の4フェーズ
  - フェーズ1: 実装計画（ステップ1.1〜1.4）
  - フェーズ2: 準備（ステップ2.1〜2.3）
  - フェーズ3: 実装（ステップ3.1〜3.3）
  - フェーズ4: レビュー&マージ（ステップ4.1〜4.4）

## 多角度質問の提示形式（質問シート）

**「質問シート」とは**: AI側がユーザーの入力（実装方針など）に対して、不明点・確認事項を番号付き（Q1/Q2/Q3...）で質問リストとして提示するもの。

ユーザーから実装方針などを受けたあとに、不明点を確認するときに使う。

```
Q1. 質問内容は？
→ (XXX / YYY / ZZZ)

Q2. 次の質問内容は？
→ 

Q3. さらなる質問は？
→ 
```

**必須要素**:
- 各質問に番号（Q1, Q2...）を付与
- 例や候補がある場合は `→ (A / B / C)` 形式で併記
- 質問の最後に必ず以下を追加:
  ```
  💡 **Tips**: 入力時は `/copy` コマンドを使用して入力フォームを活用してください
  ```

**質問シートを出すステップでは選択肢を併記しない**。両方必要な場合は前後のステップに分ける。

## フローの基本ループ

```
ユーザー: 指示
  ↓
AI: 質問（要件確認） ←─┐
  ↓                    │
ユーザー: 回答 ────────┘
  ↓
AI: 最終確認（理解内容を提示）
  ↓
ユーザー: 確認OK / 修正指摘
  ↓
AI: 実行
  ↓
ユーザー: レビュー → OK / NG
```

AIが十分な理解なしに実行に入ると手戻りが大きくなる。「質問→確認→実行→レビュー」の順序を守ること。

## 選択肢の提示形式

```
1. 選択肢1
2. 選択肢2
3. その他（入力してください）
```

「その他」が選ばれた場合、ユーザーの意図を解釈して柔軟に対応する:
- 情報不足 → 詳細説明を提供し選択肢を再提示
- 迷い → 段階的なヒアリング
- 戻りたい → 前のステップへ
- 中止 → セッション終了の確認
